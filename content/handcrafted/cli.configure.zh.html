先讲人话版

🛋 openclaw configure 像“带你一间房一间房装修”的向导

这页讲的不是底层命令手册，而是一位会问你问题的装修向导。你不需要自己记住每颗螺丝该拧到哪，它会按房间带你走。

模型、通道、设备、Gateway 运行方式，这些都像不同房间。`configure` 的工作就是带你一间间进去，把该摆的摆好。

适合想用问答方式把 OpenClaw 配顺的人 这页最重要的是记住：这是交互式向导，不是批处理扳手 查看官方原文

第一站

🏠 它最像“分房间装修”，不是一次把整栋楼拆开

官方这里最有价值的一点，是 --section。这意味着你可以只进某一个房间，不必全屋重装。

🚪

openclaw configure

像请装修向导带你从大门开始，一路把常见房间都走一遍。

🌐

--section web

像今天只改“上网这间房”，不碰别的房间。

🧠

--section model

像今天只去调脑子和模型架子。

📮

--section channels

像今天只管把消息门和门牌配好。

第二站

🧭 这页一直在提醒你：Gateway 跑在哪，不是小细节，是整栋楼的朝向

官方写得很明确，选择 Gateway 跑哪儿时，会直接改 gateway.mode。这不是枝叶，是主梁。

🏡 local

像把总机房放在自己家里。很多首次使用都会从这里开始。

🌉 remote

像总机房其实在别处，你只是过桥去控制它。

只改这一项也行

官方特地说，你甚至可以只改 Gateway 运行位置，然后就收工离开。

📌 一句话

机房朝哪放，是装修时最该先定下来的大方向。

第三站

🤖 Model 那一段，不是在堆模型名，而是在挑“脑子上架哪些按钮”

官方现在把 agents.defaults.models allowlist 放进向导里了，这其实是在说：哪些模型会出现在 /model 和选择器里，是可以控制的。

🧠 默认脑子

像先给 OpenClaw 定一颗常用主脑。

🗃 模型 allowlist

像决定哪些脑子被摆上货架，让你平时切换时看得见。

🌐 web search section

像给它再配一间侦察室，决定用哪家搜索供应商和哪把钥匙。

⚙️ Grok follow-up

如果你选 Grok，向导可能会多追问一步，像装修师傅说：“那你这间房还要不要顺手再装个侧门？”

第四站

🔐 SecretRef 和 daemon 安装那一大段，翻成人话就是“钥匙可以藏起来，但安装时必须确认它真的拿得到”

这页很强调一种稳妥风格：能用引用就用引用，但如果要把服务长期装起来，就不能装一个拿不到钥匙的门禁系统。

🪪

SecretRef 托管钥匙

像门票不直接贴在门上，而是锁进钥匙柜里。

🧪

安装前先验钥匙

如果你要装 daemon，向导会先确认那把钥匙真的能拿到。

🚫

拿不到就不硬装

官方明确会拦住。不会为了装成功，偷偷把明文钥匙乱塞进去。

📌

一句话

钥匙能藏，但长期值班前必须确认这把钥匙是真的能拿出来。

第五站

📮 通道名字和房间名字，也能边装修边帮你翻译成真正门牌号

官方提到 Slack、Discord、Matrix、Teams 这些会提示 allowlist，而且能在可能时帮你把名字解析成 ID。这很像装修时说“你写房间昵称也行，我来帮你对成正式门牌”。

🏷 输入名字或 ID 都行

像你说“客厅”或者“房号 302”，向导尽量都能帮你认出来。

🔍 能解析就帮你解析

它会尽量把模糊名字翻成系统真正认的号码。

📬 allowlist

像告诉门卫“哪些房间或频道允许收信”。

🎈 一句话

向导不只记配置，还尽量帮你把人话翻成系统话。

最后总结

🎈 把 openclaw configure 这页压成一句话

openclaw configure 就像装修向导：你可以全屋走，也可以只修某一个房间；它负责提问、翻译、校验钥匙，并尽量把复杂配置变成一步步可走的动作。

如果你下一页只想继续看一页，我建议看 /cli/config。因为当你不想走向导、想自己拧某一颗螺丝时，就该换那把工具。

先讲人话版

🏠 Configuration Examples 像“样板房大街”：不是让你全搬，而是教你怎么抄得聪明

这一页不是定义字典，也不是概念课。它就是在给你看几套现成装修样板：最小能住版、推荐起步版、展开全配版，以及几个常见场景版。

所以这页的正确读法不是从头背到尾，而是先找“哪一套最像我家”，然后先抄一小块，再慢慢改。

适合想直接抄配置的人 把样板房当模板，不要当圣经整本照抄 查看官方原文

第一套

🪑 Absolute minimum 像“先放一张桌子，先开一扇门”

官方第一套最小样板特别值得学，因为它在教你：系统能开张，不需要先塞满整栋楼。

agent: { workspace: "~/.openclaw/workspace" }

像先摆好一张工作桌。老师至少要有个地方放东西。

channels: { whatsapp: { allowFrom: ["+15555550123"] } }

像只开 WhatsApp 这一扇门，并只让一个号码先试进来。

为什么这套好

因为它让你先把“一件事能跑通”这件事做成，再说别的。不是一上来把所有房间都装修。

第二套

🌟 Recommended starter 像“可以真正住进去的第一版小房子”

这一套在最小版基础上，多了身份、默认模型和群聊提及规则，所以更像可长期住的起步房。

🙂

identity

像给你的机器人起名字、定人设、选表情，不再只是一个没脸没名的小黑盒。

🧠

model.primary

像明确请一位默认老师长期坐班。

👆

groups: { "*": { requireMention: true } }

像写一块牌子：“群里请先点名，再找我说话。”

🎈

这一套适合谁

大多数第一次正式搭自己的 OpenClaw，都该从这套气质出发。

第三套

🏘 Expanded example 像“看一整条样板房街”

这部分很长，但别硬吞。它不是让你整块照搬，而是在展示“楼里还能加哪些房间”。

🔐 env / auth

像钥匙间和门禁册。不同 provider 的钥匙、登录档案，放在这类区块里。

🪪 identity / logging / messages

像门牌名、监控记录、说话风格这些基础生活设施。

🧰 tools / media / sandbox

像工具间、影音室、隔离实验室。高级一点，但都很实用。

🚦 routing / session / gateway / hooks / cron

像楼里的交通规则、记忆规则、总开关室、门铃系统和定时广播系统。

第四套

🌉 Multi-platform setup 像“同一位老师开了好多扇门”

WhatsApp、Telegram、Discord 一起上，不是在变复杂，而是在说：同一位 OpenClaw 可以同时守多扇门。

但样板也顺手在提醒你：不同门的 ID 规则都不一样。手机号、用户 ID、频道 ID，别拿错门牌号。

第五套

🔒 Secure DM mode 像“不要让不同来访者共用同一本谈话笔记”

这一套特别重要，因为它讲的是多用户私聊时最容易踩的大坑。

session: { dmScope: "per-channel-peer" }

像给每个来访者单独准备一本笔记本。这样 A 来问的问题，不会和 B 的上下文搅在一起。

什么时候一定该开

只要不止一个人能私聊你的 bot，就该优先考虑这套。共享 DM 上下文很容易串味。

🎈 最短记法

多人 DM，不要共用一本谈话本。

第六套

🔁 OAuth + API key failover 像“先刷会员卡，不行再掏备用钥匙”

这一类样板的价值，不是字段本身，而是思路：一条路不通时，系统可以换下一条认证路继续走。

🪪 OAuth 在前

像先试账号票，省事、顺手。

🔑 API key 在后

像会员卡刷不过时，再掏正式钥匙补上。

auth.order

像在门禁本上写明“先试哪张卡，再试哪把钥匙”的顺序表。

第七套

🏢 Work bot / local models only，这些都是“不同房型”

后面几套看着五花八门，其实只是不同用途的房型样板。

💼

Work bot

像办公室用房。门更少、权限更严、语气更专业。

🏠

Local models only

像完全在自己家里做饭，不叫外卖。全部靠本地模型厨房。

🧪

Anthropic + fallback 例子

像主厨是 Claude，备选厨师是别的平台模型。

🎈

真正要学的

不是背每一套，而是学会“从哪一套开始抄最省力”。

最后总结

🎈 把配置样例页压成一句最好记的话

这一页是样板房大街。先找最像你家的那一套，抄最小可用的一块，先住进去，再慢慢加家具。别把整条街一起搬回家。

先讲人话版

🚚 Migration Guide 这页，其实是在教你“给 OpenClaw 搬家，但别把记忆、钥匙和门牌落在旧房子里”

这页最重要的观念只有一个：不是只搬一个 openclaw.json 文件，而是把整个状态目录和工作区完整搬走。

如果你只拿走表面那张配置纸，很多真正重要的东西，比如凭据、通道登录、会话状态，都会留在旧房子里。

适合迁移到新机器 重点是状态目录、workspace、doctor、同 profile 查看官方原文

第一站

📦 真正要搬走的，不是一张配置纸，而是一整箱生活用品

官方列的 “What Gets Migrated” 很值得直接压成画面感。

⚙️

配置

像房子的规则表，告诉 Gateway 怎么工作。

🔑

凭据和授权

像各种门卡、钥匙、OAuth 通行证。

💬

会话和通道状态

像聊天记录和“这扇门已经登录好了”的现场状态。

🗂️

工作区

像记事本、技能卡、提示词文件，全都在这里。

第二站

🛑 搬家第一步不是复制，而是“先让旧房子里的值班员停下来”

不然边搬边改文件，很容易搬出半旧半新的怪状态。

openclaw gateway stop

像先让旧房子的值班员下班，别在你打包时还一直改抽屉里的东西。

tar -czf openclaw-state.tgz .openclaw

像把整个状态箱打成一个搬家包，而不是零零散散抓几个文件。

📌 多 profile 怎么办

如果你有 .openclaw-work 这类多个房间，就每个房间各打一包，别混成一团。

🎈 一句话

先停，再打包，别边跑边搬。

第三站

📥 新机器那边的动作，其实是“先铺一层新地板，再把旧家当整个盖上去”

官方说 onboarding 生成了新目录也没关系，就是这个意思。

🆕 先装 CLI

像先让新房子具备基本用电和开门能力。

scp / rsync -a

像把搬家箱从旧房子送到新房子。

tar -xzf openclaw-state.tgz

像在新房里把整箱家当打开，按原样摆回去。

📌 隐藏目录别漏

这也是为什么官方特别提醒“确保 hidden directories 也被带上”。很多关键东西就藏在那里。

第四站

🩺 最后一定要 doctor，因为它像搬家后的总验收师

它不只是“看有没有活着”，还会顺手补迁移和修细节。

openclaw doctor

像请总验收师来检查：配置迁移好了没，路径修对了没，服务状态顺不顺。

openclaw gateway restart

像让值班室在新房里正式重启上岗。

openclaw status

最后看灯牌是不是亮了，别以为搬完箱子就算入住成功。

🎈 收尾一句

这页最重要的提醒是：搬家要搬整箱，不要只拿封面那张配置纸。

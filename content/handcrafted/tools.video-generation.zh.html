先讲人话版

🎬 `video_generate` 像给代理一台会剪片的后台工厂

它能把文字、图片、视频参考变成视频，而且是异步做完再回传成品。

这页最重要的是：视频生成不是单步动作，而是有运行模式的长工单。

适合想让代理做视频的人 重点是 generate / imageToVideo / videoToVideo 查看官方原文

第一站

🧭 先认出三种工作模式

它们对应不同的输入材料。

A

generate

纯文字生视频。

B

imageToVideo

拿图做视频。

C

videoToVideo

拿视频再加工。

🧷

模式先验

工具会先验证当前模式是否支持，再提交。

第二站

⏳ 视频是长工单，先提交，再回来取结果

这和普通工具瞬时返回不一样。

提交任务

先拿到 task ID。

后台生产

通常要等一会儿。

完成回流

结果再被唤回原会话。

重复调用

同一会话里再点一次，多半只是查状态。

第三站

🛠 支持的 provider 很多，但你先别全背

先知道它支持很多视频后台就够了。

Google / OpenAI / Runway / Vydra

只是列表里的一部分常见老师。

默认模型

可以从配置里钉住。

asynchronous

视频生成本身就是异步流程。

最后总结

🎈 把 Video Generation 这页压成一句最好记的话

`video_generate` 是视频工厂：先选模式，再提交长工单，最后等成片回到会话里。

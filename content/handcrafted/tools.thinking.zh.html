先讲人话版

🧠 Thinking Levels 就是给 OpenClaw 的脑子拧一个“想多一点还是想快一点”的旋钮

这页看起来像在讲 /think，其实是在讲一件很简单的事：同样是回答问题，OpenClaw 可以选择“先想一下就说”，也可以选择“多想一会儿再说”。

你可以把它想成给脑袋换挡。不是换人，只是换这一次的思考速度和深度。

适合想控制模型思考深度的人 这页最重要的是分清“这一次”和“整个会话”不是一回事 查看官方原文

第一站

🎚 先把这事想成“给脑子调档位”

官方一上来列了一堆 level 名字，别先背。先想画面就够了：你是在调 OpenClaw 这一回合要用多大的脑力。

🐣

off

像把思考加成关掉。让它别特意多想，直接轻快一点。

🪶

minimal / low

像先小想一下再开口。适合不需要太多绕路的时候。

🪫

medium

像认真一点，但还没到把整本作业本都翻一遍的程度。

🔥

high / xhigh

像把脑力档位往上拧。high 是重想，xhigh 是更高一档的重想，主要给更吃脑力的模型用。

第二站

🏷 这些名字像口令，但它们背后只是“这次要多用几分脑子”

官方还给了很多别名。你不用把它们当术语看，直接当成“同一档位的不同叫法”就行。

🧩 x-high / x_high

这几个都只是 xhigh 的不同写法，像同一把门匙换了几种写法。

🏁 highest / max

这两个都映射到 high。像你说“拉到最高”，系统就帮你翻译成它真正认识的那个档位。

🪄 adaptive

像把决定权交给模型家长自己。尤其 Anthropic Claude 4.6 会默认往这个方向走。

⭐ 先记一句

Thinking levels 不是新功能，而是同一件事的不同脑力强度。

第三站

✏️ /think 写在消息里时，像给这一回合贴一张小纸条

官方很认真地区分了三种写法。别把它们想成三种不同魔法，其实都是在告诉系统：这次想法档位要怎么调。

/t high

像你直接说：“这次请多想一点。”

/think:medium

像你把那张纸条贴得更明确一点：“这次中等认真。”

/thinking low

也是同一件事，只是换了一个入口。像同一扇门的三个门铃按钮。

🧠 单条消息和会话默认不一样

如果它只是夹在一句普通话里，那更像只影响这一次；如果整条消息只有这个 directive，才更像把整段会话的默认档位改了。

第四站

📏 这个档位到底先听谁的

官方写了一长串优先级。翻成人话，就是“越靠近这一刻、这一条消息、这一个代理的设置，越说了算”。

1. 当前消息里的指令

最先生效。像你刚刚当场举手说的话，谁都改不了。

2. 会话里已经存好的默认值

像这间教室已经挂着的一块牌子。没新指令时，就照这块牌子来。

3. 当前代理自己的默认设置

像这个老师自己平时的讲课风格。

4. 全局默认和兜底

如果前面都没说清，系统才会往更大的默认规则里找答案。像最后再看学校总规章。

第五站

🧷 设成“只影响这一回”还是“以后都照这个来”，差别很大

官方特别提醒：有些指令只是这次生效，有些则会变成会话默认。这个差别会直接影响你后面每次说话的味道。

/think:medium 单独发

像把一张便签贴在这一回合上，做完就先不管。

只发 /think:medium

像你专门对教室说：“从现在起，这间教室的默认脑力档位就这样。”

/think off

像把那张默认纸条撕掉，回到最朴素的状态。

/think 什么都不写

像抬头看看现在到底挂着什么牌子。官方说它会告诉你当前 thinking level。

第六站

🧠 不同模型，对“想法档位”的脾气也不一样

这页不是单纯在讲名字，它还在提醒你：不同提供商对 thinking 的支持方式不同。

🤖 Claude 4.6

默认会走 adaptive。像系统自己觉得该想多少就想多少。

🟢 Z.AI

它只认“开”或“关”两种状态。你给它别的档位，它会把它理解成开脑子或关脑子。

🌙 Moonshot

它对 /think off 的理解也很直接：就是关掉思考。开着的时候，它还会对 tool choice 有自己的规矩。

📌 这节真正的意思

不是所有模型都把“思考档位”理解成同一件事。你得知道它们各自爱怎么接这个按钮。

第七站

⚡ Fast mode、Verbose、Reasoning，都是在管“要不要把后台声音放出来”

官方在这一页后半段还顺手讲了几个相邻开关。它们和 thinking 一起，像一组控制面板。

⚡

/fast

像把车切到快一点的档位。重点是速度和响应感，不是“想得多不多”。

🗣

/verbose

像让后台多说一点给你看。开了以后，工具调用和调试痕迹会更明显。

📣

/reasoning

像把“它在脑子里想了什么”单独端出来。开着时，理由会变成单独的消息。

🧵

顺序很重要

这些开关会按消息、会话、代理默认值一路往下找，不是你随便一开就永远生效。

第八站

🌙 Heartbeat 和 Web UI，只是在说“默认值会不会被别的地方带着跑”

官方最后还补了两个小地方：心跳消息和网页选择器。它们不是主角，但能帮你理解系统怎么记住默认值。

💓 Heartbeats

像系统定时来拍你一下，确认工作区里还有什么要紧事。里面的指令照样有效，但最好别拿它来改长期默认。

🖥 Web chat selector

像网页里有一个临时换挡器。它只影响下一条消息，发完以后又会回到会话默认值。

🎈 一句话收住

Thinking levels 这页不是在讲“模型会不会思考”，而是在讲“这一次让它想多深、之后默认记住哪一档”。

先讲人话版

🧭 Northflank 这页，其实是在教你“点一个托管模板，把 OpenClaw 交给云平台值班”

它和 Railway 很像，都是“服务器上不碰太多终端”的轻托管路线。你最需要补的，不是构建命令，而是那把管理员门票。

所以官方流程看起来短，是因为平台帮你包掉了大部分基础设施杂活。

适合轻托管部署 重点是一键模板、OPENCLAW_GATEWAY_TOKEN、/openclaw 查看官方原文

第一站

🚀 开始步骤其实只有一件大事：把模板点起来，再把门票填进去

它短不是因为没内容，而是因为平台替你做了很多后台动作。

Deploy OpenClaw 模板

像去前台领一套现成装修包，房子的基础框架不用你自己画。

OPENCLAW_GATEWAY_TOKEN

这是这条路最重要的手动输入。没有它，控制台就像有门没锁，或者反过来你自己也进不去。

View resources

像部署结束后去看房产交付清单，确认服务真的已经起来。

/openclaw

控制台入口在这里，不是打开根域名随便乱点。

第二站

💾 平台帮你做的最大好事，是把持久存储也打包好了

这点和 Railway 一样重要。

📦 Northflank Volume

像给 OpenClaw 单独配了一只储物柜，不会因为重部署就把记忆和凭据丢掉。

🖥️ Hosted Gateway + Control UI

像前台和后台都有人帮你看着，不需要你自己先 SSH 进去搭整间房。

🎈 一句话

Northflank 这条路本质上就是“平台托管 + 你自己握住门票”。

第三站

📲 后面真正要做的，就是接通道和继续日常运维

部署成功只是把房子盖好，不等于已经有客人能进门聊天。

📡 先接 Telegram / Discord

像先装一两扇最常用的门，让 OpenClaw 真正能开始接消息。

🧰 还可以 SSH 进去跑 onboarding

平台虽然主打轻托管，但必要时你还是能走维修通道进去操作。

🎈 收尾一句

Northflank 这页最像“托管式开店”，不是“从地基开始盖房”。

先讲人话版

🧪 Synthetic 像“一个披着 Anthropic 外套的模型货架”

官方这页一上来就说得很直：Synthetic 对外看起来像 Anthropic 兼容接口，所以 OpenClaw 会按 Anthropic Messages 这套方式去跟它说话。

适合想走 Synthetic 这条 Anthropic 兼容路的人重点是兼容外壳、默认 MiniMax、模型货架很杂查看官方原文

第一步

🔑 接法很简单：给 SYNTHETIC_API_KEY，默认先拿 MiniMax 那个货架

主路非常短。你给 key，跑 onboarding，默认模型就先落到 synthetic/hf:MiniMaxAI/MiniMax-M2.5。

这条默认模型名字看起来怪，是因为它像在写“Synthetic 商场 / Hugging Face 样式货号 / 具体模型”。不是故意绕，而是在写完整路径。

第二步

🧺 这页真正特别的地方，是一个货架里摆了很多别家模型

官方给了一长串模型目录。你别逐条背，把它看成“同一个外壳下可以挑很多货”。

MiniMax / Kimi / GLM / DeepSeek / Llama / Qwen

像一个混合品牌大货架，不是只有一家自营。

全部成本先写 0

像货架价格牌先没细填，侧重的是“可接入”，不是精确计费展示。

📌 真正要记住的

如果你启用了 allowlist，别忘了把你真想用的每个模型都加进去。不然货在架上，你也拿不到。

第三步

🧱 baseUrl 那句提醒，像“门牌号不要多写一层楼”

官方专门提醒不要把 /v1 再手动写进去，因为 Anthropic 客户端自己会补。

所以这里最像“门牌号别写重复楼层”。写多一层，车就会开错门口。

最后总结

🎈 把 Synthetic 这页压成一句最好记的话

Synthetic 是披着 Anthropic 外套的模型货架。主路很短，但货架很杂，真正要小心的是模型 allowlist 和 baseUrl 门牌别写重。

先讲人话版

🎛 Models 像“给 OpenClaw 排主脑、备胎脑和图片脑的控制台”

这页最重要的不是模型名字，而是顺序。谁先上场，谁在后面候补，遇到图片时换谁，这些都在这里讲。

你可以把它想成脑子调度台。

适合想真正配清主模型、fallback 和 allowlist 的人 这页最重要的是记住：主脑、后备脑、图片脑是三条不同线 查看官方原文

最先记顺序

🥇 OpenClaw 选脑子的顺序，其实像“一队主力 + 替补 + 供应商内部再换卡”

官方把顺序写得很清楚，这一段就是整页的骨头。

1

Primary

像首发主力，先让它上场。

2

Fallbacks

如果主力不行，就按顺序把替补一个个叫上来。

3

Provider auth failover

在正式换到下一颗模型前，这家供应商自己内部还可能先试别的认证卡。

📌

一句话

先换卡，再换脑子；先主力，再替补。

三条线

🧠 主脑、🖼 图片脑、🎨 图像生成脑，不是一回事

官方把这几把钥匙分得很清楚，这很值钱。

agents.defaults.model

像平时最常坐在前台的那颗主脑。

agents.defaults.imageModel

像主脑看不懂图片时，临时请来的图片翻译员。

agents.defaults.imageGenerationModel

像专门负责画图的画师，不等于聊天主脑。

🎈 记一句

能聊天的，不一定能看图；能看图的，不一定会画图。

Allowlist

📚 agents.defaults.models 像“允许上架的脑子目录”

这是最容易把人绊住的一块。因为一旦你写了它，OpenClaw 就会把它当作 allowlist。

📚

写了就生效

不是装饰目录，而是真正的“允许上架名单”。

🚫

Model is not allowed

像有人想从货架外拿一颗没登记的脑子，前台会直接拦住。

🪄

Alias

像给长长的模型全名再贴个短昵称，日常切换时更顺口。

🎈

一句话

写了 allowlist，就别忘了把你真想用的脑子都放进去。

命令翻译

🛠 这些 openclaw models ... 命令，像脑子调度台上的按钮

别把它们当成孤零零的命令。它们其实是一排很实用的控制按钮。

openclaw models list

像打开货架目录，看看现在到底都摆了哪些脑子。

openclaw models status

像看前台值班表：主力是谁，替补是谁，认证卡状态又怎么样。

openclaw models set <provider/model>

像直接改掉首发主力，把下一场先发换成另一颗脑子。

/model、/model list、/model status

像在聊天现场边说边换脑子，不必回机房手改总账本。

最后总结

🎈 把这页压成一句最短的话

Models 讲的就是脑子调度台：谁是主脑、谁是替补、谁专门看图、哪些脑子准许上架，以及你该按哪个按钮临时切换。

先讲人话版

🦞 Agent Runtime，就是 OpenClaw 里面那位“真正干活的小工人”

很多人会把 OpenClaw 想成一个大盒子。其实盒子里最重要的，是那位一直在收消息、想办法、调工具、再把结果送回去的小工人。

这页讲的，就是这位小工人平时住哪、看哪些纸条、能用哪些工具、又有哪些边界。

适合想弄懂 OpenClaw 里面到底是谁在干活的人 这页最重要的是看懂：工作区是家，bootstrap 文件是开工前要看的小纸条 查看官方原文

第一眼

🏠 工作区不是“随便一个文件夹”，而是小工人的家

官方开头就讲得很直白：Agent 只有一个主要工作目录。你可以把它想成小工人的家和书桌。

🏡

agents.defaults.workspace

这就是小工人的家。它平时读文件、找上下文、动工具，默认都从这里出发。

🧰

工具默认从这里开工

就像工人从自己的工作台上拿尺子、螺丝刀，不会先跑去街上乱翻。

🪟

沙箱像临时工棚

如果开了 sandbox，某些会话会被带到另外一间受控工棚里干活，不直接碰主屋。

📌

一句话

工作区就是它的家。家里摆什么，直接决定它上班时先看到什么。

开工前

📚 那几份 bootstrap 文件，像桌上早就摆好的开工小纸条

这部分最值得读。因为它解释了为什么 OpenClaw 刚开口时，看起来像“已经知道一些事”。

AGENTS.md

像“工作守则”。告诉它平时做事的规矩和优先级。

SOUL.md

像“性格说明卡”。决定它说话风格、边界感、气质像什么。

TOOLS.md

像“本地工具使用备忘录”。不是发工具许可证，而是告诉它这些工具在你家怎么用更合适。

USER.md / IDENTITY.md

一个像“你是谁”，一个像“它是谁”。这样它开口时不会连主人和自己都认错。

🧾 这些纸条什么时候塞进脑子里？

官方说得很清楚：新会话第一轮开始时，OpenClaw 会把这些文件内容直接注入上下文。你可以把它想成“小工人上岗前先把桌上的便签扫一眼”。

命令翻译

🛠 openclaw setup 在现场到底做了什么？

官方推荐在缺东西时跑 openclaw setup。这条命令不是魔法，它更像“先把家搭起来”。

openclaw setup

像搬家师傅进门，先把默认配置本、工作区和那几张基础便签摆好。不是替你写人生规划，而是先把桌椅灯装齐。

缺文件时的安全默认值

如果某张便签没了，OpenClaw 会补一个安全模板，不会让小工人直接对着空房间发呆。

skipBootstrap

像你提前把房间布置好了，于是告诉搬家师傅：“别再给我默认摆设了，我自己来。”

🎈 记一句

setup 干的是搭家底，不是正式上班。

工具和技能

🧰 工具是它手里的家伙，Skills 是它临时翻的小册子

这页有个很容易误会的点：TOOLS.md 不决定工具存不存在。工具和说明书，是两回事。

🔧

Built-in tools

像工人本来就有的锤子、钳子、扳手。是否能用，还要看工具策略和配置。

📓

TOOLS.md

像你贴在墙上的“这些工具在我家怎么用比较合适”的小提示。

🪄

Skills

像遇到特别任务时，再去抽屉里翻专门小册子。工作区里的技能册优先级最高。

📚

三个来源

官方说 skills 可来自内置、用户管理目录、工作区。离你家越近，声音越大。

会话和边界

🧵 Session 是一本本聊天工作本，边界则像“这不是 OpenClaw 自己发明的全部世界”

官方这里有两句很重要。第一句是会话会被存起来。第二句是底层 agent core 和 OpenClaw 自己的外层工作要分开看。

📒 Session transcript

每个会话都像一本工作本，存在 ~/.openclaw/agents/<agentId>/sessions/ 下面。

🦴 Pi agent core

像真正会思考、会调工具的骨架。

🦞 OpenClaw 外层

像把这副骨架接上聊天软件、会话管理、发现机制、路由和配送系统。

📌 一句话

小工人会被记录、会被管理、会被配送，但干活那颗核心脑子并不是整座楼的全部。

最后总结

🎈 把这页压成一句最短的话

Agent Runtime 就是 OpenClaw 里的那位真工人：住在工作区，开工前先看桌上便签，手里拿工具，必要时翻技能册，边干边把每次会话写进自己的工作本。

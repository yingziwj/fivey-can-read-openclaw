先讲人话版

🐋 ClawDock 像 Docker 安装版里的“快捷按钮层”

它不是另一个大系统，而是一层 shell 小帮手，把一串很长的 docker compose 命令，变成短短几句更好记的命令。

如果 Docker 已经装好了，ClawDock 会让你少敲很多重复句子。

适合 Docker 安装路线的人 重点是 helper 脚本、dashboard、logs、token 修复 查看官方原文

第一步

🧷 先装 helper 脚本，像给 Docker 家族装遥控器

官方给的是一条 canonical path，意思是走新路，别再用旧脚本了。

clawdock-helpers.sh

像遥控器的说明书兼按钮板。

source ~/.zshrc

像把这排按钮真正接到你的终端里。

旧路径被移除

像提醒你：以前那条旧走廊已经拆了，不要再过去敲门。

第二步

🛠 这些短命令，都是在替你代敲很长的 Docker 指令

ClawDock 的价值就在于把“很长很重复”变成“短而稳”。

▶

clawdock-start

像一键把 Gateway 开起来。

⏹

clawdock-stop

像一键收工。

📺

clawdock-dashboard

像直接打开网页控制台看总控室。

🪛

clawdock-fix-token

像给门禁卡重新补钥匙。

第三步

📦 它本质上只是 Docker 装法里的一个小便利层

如果你还没装 Docker，官方还是建议先去 Docker 那页。

Basic ops

启动、停止、查看状态、跟日志，都是最基本的值班操作。

Container access

像直接钻进容器里看现场。

Web UI

像从浏览器里看这一整套系统。

最后总结

🎈 把 ClawDock 这页压成一句最好记的话

ClawDock 就是 Docker 安装路线的快捷按钮层：把长命令压短，让你启动、看板、查日志、修 token 都少敲很多字。

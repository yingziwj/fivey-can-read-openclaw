先讲人话版

Quickstart 是 ClawHub 的“第一次逛货架路线图”

这页不想把你塞进一堆概念里，而是带你做几件最常见的小事：找 skill、装 plugin、登录、发布。

你可以把它当成游乐园门口的小地图：先知道入口在哪里，再决定今天要不要玩更难的项目。

适合刚开始试 ClawHub 的人 重点是按顺序跑通第一套动作 查看官方原文

第一站

先找一个 skill，再把它装进当前工作区

skill 像一张教 OpenClaw 做某件事的小卡片。你先搜索，确认合适，再安装。

openclaw skills search "calendar"

像在书架前找“和日历有关的小课本”。

openclaw skills install <skill-slug>

像把小课本放进你的书包，下一次上课就能拿出来用。

装完后

新会话通常更容易看到新装的 skill，因为它像重新发了一遍课本。

第二站

plugin 是更大的能力盒子，安装前更要看清名字和来源

plugin 不只是说明书，它可能带来新的工具、通道、模型供应商或运行能力，所以更像能插进机器里的零件。

openclaw plugins search "calendar"

先找货，不急着装。

openclaw plugins install clawhub:<package>

确认是想要的那盒，再拿回家。

重启或重新加载

很多 plugin 装完后需要让 Gateway 重新看一遍货架，才会真正开始工作。

发布者路线

想把自己的东西放上架，就要登录并准备好一个规范的文件夹

发布 skill 时，核心是一个带 SKILL.md 的文件夹。发布 plugin 时，通常像 npm 包一样，要有清楚的名字、版本和入口。

1

安装 CLI

发布和登录属于“店主动作”，需要 clawhub CLI。

2

登录

像证明“这盒东西是谁放上来的”。

3

发布

上传文件、版本和说明，ClawHub 才能把它整理到货架上。

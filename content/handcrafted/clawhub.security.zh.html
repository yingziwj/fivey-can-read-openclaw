先讲人话版

ClawHub Security 是货架门口的“先检查再拿走”

开放货架很好，因为大家都能分享好东西。但开放也意味着要小心：盒子里可能有不安全的指令、奇怪的文件，或不适合装进系统的行为。

这页讲的是用户能看什么、扫描状态怎么理解、举报怎么处理，以及为什么安装前要先看标签。

适合安装第三方包前先看一遍 重点是扫描、可见信息、举报和 moderation 查看官方原文

第一站

安装前先看三张小票

ClawHub 页面会给你一些线索。它们不能保证万无一失，但能帮你少踩明显的坑。

来源

谁发布的？是不是你信任的 owner？名字有没有冒充别人？

版本和文件

最近有没有更新？文件看起来是不是和功能相符？

扫描结果

有没有被标记出需要注意的地方？不要看到警告还闭眼安装。

第二站

skill 和 plugin 的检查重点不一样

skill 更像说明书，检查重点是它要求模型做什么、声明了什么运行需求。plugin 更像能运行的零件，检查重点还会包括包信息、入口、兼容性和产物完整性。

S

Skill

看指令有没有危险要求、metadata 有没有说清需求、内容有没有可疑行为。

P

Plugin

看包来源、入口、兼容字段、发布记录和完整性信息。

R

Report

发现问题可以举报，理由要写清楚，像给管理员递一张说明纸。

最后记住

安全不是吓人，而是让开放货架可以长期热闹

ClawHub 的扫描、举报和隐藏机制，都是为了让好东西更容易被放心使用，也让坏东西更快从显眼位置退场。

先讲人话版

clawhub CLI 像仓库后台的遥控器

普通安装技能时，多数时候用 openclaw 就够了。clawhub CLI 更像给发布者、维护者和自动化脚本用的后台遥控器。

它负责登录、确认身份、搜索、发布、撤回、同步和管理包。也就是说，它不是“拿货小车”，更像“货架管理员的工作台”。

适合要发布或维护 ClawHub 包的人 重点是登录、配置、发布和安全操作 查看官方原文

第一站

全局参数像“先告诉遥控器你站在哪里”

CLI 里很多参数不是在做大事，而是在说清楚工作目录、安装目录、站点地址和输出格式。

--workdir <dir>

像说：“今天我们在这张桌子上干活。”

--dir <dir>

像说：“拿回来的 skill 要放进哪个抽屉。”

--json

像把结果写成机器也能读懂的表格，方便脚本接着处理。

第二站

登录像拿到店主钥匙，不能随便乱放

clawhub login 会把 API token 保存到配置里。这个 token 像钥匙，能证明你是谁，也可能允许你发布东西。

1

浏览器登录

适合本机有浏览器时使用，像走正门刷卡。

2

token 登录

适合服务器或脚本，但 token 要像钥匙一样保管。

3

device 登录

适合远程环境，CLI 给你一个码，你去别处完成授权。

4

whoami

像问系统：“我现在拿着的钥匙到底是谁的？”

第三站

发布、撤回、同步，都是“管货架”的动作

这些命令会影响公开货架上的内容，所以要比普通搜索和安装更谨慎。

publish

把本地文件夹或包送上货架，让别人能找到。

yank

像把某个有问题的版本从显眼位置撤下来，避免更多人继续拿。

sync

像重新整理货架索引，让 ClawHub 和来源保持一致。

先讲人话版

🏬 Provider Directory 像“给 OpenClaw 选脑子供货商的大卖场”

聊天通道讲的是“消息从哪扇门进来”，Provider 讲的是“脑子和模型从哪家供货商拿”。这页就是总入口。

你要做的事其实很简单：先挑一家供货商，给它门卡，再把默认模型写成 provider/model。

适合第一次给 OpenClaw 选模型供应商的人 这页最重要的是记住：通道是门，provider 是脑子来源 查看官方原文

第一眼

🧠 这页只讲“脑子供货商”，不讲 WhatsApp、Telegram 那些聊天门

官方特意提醒了这一点，因为很多人会把 provider 和 channel 混在一起。

🚪

Channels

像门。决定消息从哪儿进来、从哪儿送回去。

🧠

Providers

像脑子供货商。决定 OpenClaw 背后真正用哪家模型说话。

🏷

provider/model

像给每颗脑子挂上的完整门牌。前半段是谁家供的，后半段是哪一颗。

📌

一句话

先挑脑子卖场，再选货架上的具体型号。

最短上手

🪜 官方给的 quick start，其实就是两步

不用多想，先把最短路径记住。

1. Authenticate

像先去这家卖场办会员卡，让 OpenClaw 有资格进去拿货。

2. Set default model

像在总账本上写清“以后默认先拿这颗脑子”。

{ agents: { defaults: { model: { primary: "anthropic/claude-opus-4-6" } } } }

这段配置翻成人话就是：“默认主脑，先用 Anthropic 家那颗 Opus。”

🎈 记一句

先办卡，再挑一颗默认脑子，够开始用了。

这座卖场里有什么

🛒 官方列的长清单，不是要你全背，而是告诉你“这家店货很多”

OpenAI、Anthropic、DeepSeek、Google、OpenRouter、Ollama、Bedrock、xAI……这些都只是不同货架。

🌟

大厂直供

像 OpenAI、Anthropic、Google 这些，直接从原厂拿货。

🛍

聚合卖场

像 OpenRouter、LiteLLM、Cloudflare AI Gateway，一家店里摆很多家货。

🏠

本地和私有路线

像 Ollama、vLLM、SGLang，更像你把模型货架搬到自己家里。

🎤

专用能力供应商

比如 Deepgram 这种更偏转写能力的供货商，和通用聊天脑子不是同一类货。

最后总结

🎈 把这页压成一句最短的话

Provider Directory 就是模型供货商总入口：先办认证卡，再把默认脑子写成 provider/model，剩下只是挑哪家货架更适合你。

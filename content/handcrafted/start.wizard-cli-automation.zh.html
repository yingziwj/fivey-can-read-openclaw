先讲人话版

🤖 CLI Automation 像“把向导老师变成脚本机器人”

这页讲的是：如果你不想一题一题点菜单，也不想坐在终端前慢慢答题，就可以让 openclaw onboard 自动跑起来。

它不是偷懒，它是把“我已经知道要填什么”这件事，交给脚本直接做完。

适合要批量部署、写脚本、重复装机的人 这页最重要的是记住：--json 不是自动化开关，--non-interactive 才是 查看官方原文

先记住

🧯 真正让老师闭嘴的，是 --non-interactive

很多人会误会 --json 就是静默模式。不是。--json 只是把结果写成机器能读的形状。

🧠

--non-interactive

像告诉老师：“今天别一题一题问我，我已经把答案写好了。”

🧾

--json

像把最后结果装进清单盒子，给机器看得懂，但它不会让流程自动安静。

📁

--workspace

像告诉脚本：这次要在哪个家里开工，别乱跑到别的房间去。

🔐

--secret-input-mode

像决定钥匙是写在纸上，还是写成“去某个抽屉拿”的引用。

基础剧本

🎬 最短的一条自动化路线，像把开荒脚本写成一张小条子

官方先给了一个标准例子。它做的事其实很朴素：选本地模式、填认证、设门牌、装后台值班员、跳过技能步骤。

openclaw onboard --non-interactive ...

整条命令像一张工地施工单。每个参数都像在告诉老师：“这一格我已经决定好了。”

--mode local

像说这次总机房就在本机，不去远端借房。

--auth-choice apiKey

像直接指定今天用哪把钥匙，不再现场讨论。

--install-daemon

像顺手请一个后台值班员，开机以后自动上岗。

钥匙怎么放

🔑 这个页面最有用的地方，是告诉你钥匙能放成两种样子

一个是直接放明文，一个是放成引用。自动化时，这个选择很关键。

✍️

plaintext

像把钥匙直接写在单子上。简单，但要自己看住。

🗂

ref

像不写钥匙本身，只写“去哪个抽屉、拿哪把钥匙”。

🧪

env / file / exec

像钥匙柜可以来自不同地方：环境变量、文件，或者一台取件机。

⚠️

别漏环境变量

如果你选了引用模式，脚本环境里要真的能拿到那把钥匙，不然它会直接停下。

不同例子

🧩 官方放了一排例子，像给你看“同一张剧本怎么换演员”

这些例子不是让你背下来，而是让你知道：不同 provider 都能这样被脚本化。

Gemini / Z.AI / Vercel / Cloudflare / Moonshot / Mistral / Synthetic

像给不同店铺分别发施工单，但每家店的门牌写法略有不同。

OpenCode / Ollama / Custom provider

有的是现成柜台，有的是本地柜台，有的是你自己搭的柜台。

--custom-base-url / --custom-model-id

像给自定义供应商写地址和房间号。

--custom-compatibility

像告诉老师：“这家店说话风格更像 OpenAI 还是 Anthropic。”

多一个助手

🧑‍🤝‍🧑 这页还教你：如果要再开一位小助手，脚本也能一起做

OpenClaw 不止能装一次，脚本也能顺手把新的 agent 一起创建出来。

openclaw agents add work ...

像给新助手起名字、分房间、分工作台，再把它接进系统。

--workspace ~/.openclaw/workspace-work

像告诉它新房间摆在哪。

--bind whatsapp:biz

像提前把它接到某个消息门上。

📌 一句话

自动化不只是省时间，它也让你能把“多开几位助手”写成脚本。

最后总结

🎈 把这页压成一句最短的话

CLI Automation 就是把向导老师变成脚本机器人：用 --non-interactive 让它别问东问西，用 ref 把钥匙藏好，再按不同 provider 的剧本批量装机。

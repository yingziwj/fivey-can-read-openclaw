先讲人话版

🧠 Anthropic 这页，其实是在问：你想拿钥匙进 Claude 家，还是直接把家里的 Claude 柜台接进来？

官方这一页看起来东西很多，其实只有三条主要路。第一条是 API key，像拿正式钥匙进门。第二条是 setup-token，像拿订阅门票。第三条是 Claude CLI，像你家里已经有一个 Claude 柜台，OpenClaw 直接过去接线。

后面那些 thinking、/fast、缓存、1M 上下文，看着吓人，其实都只是“门已经开了以后，这位 Claude 老师要怎么工作”。

适合正在接 Claude 的人 先选进门方式，再决定要不要开加速、缓存和长上下文 查看官方原文

第一步

🚪 先分清三条路，不然后面会越看越乱

Anthropic 这页最容易把人绕晕的地方，就是把三条路线混在一起看。其实它们各管各的。

🔑 API key

像正规的公司钥匙。最稳，最适合长期值班，按量计费也最清楚。

🎟 setup-token

像 Claude 订阅票。能进门，但更像“拿票刷卡”，不是万能总钥匙。

🖥 Claude CLI

像你家电脑上已经有 Claude 柜台，OpenClaw 不再自己去 API 大楼排队，而是直接请这台本地柜台出手。

🎈 一句话收住

API key 是正式钥匙，setup-token 是订阅门票，Claude CLI 是把本地柜台接进来。

第二步

🔑 如果你走 API key 路线，重点就是“把钥匙放稳”

官方其实最推荐这条给长期运行的 Gateway。因为它最直接，最像一把固定门卡。

🧭

适合谁

想稳定跑、想自己看账单、想少一点奇怪门禁的人，先走这条。

🛠

openclaw onboard

像带你走一遍登记台，最后选择 “Anthropic API key”。

⚡

openclaw onboard --anthropic-api-key "$ANTHROPIC_API_KEY"

像你已经把钥匙握在手里了，不走问答，直接塞给系统。

📦

env: { ANTHROPIC_API_KEY: "sk-ant-..." }

像把钥匙放进固定钥匙盒，方便 Gateway 以后自己拿。

第三步

🤔 thinking 在说什么？它像 Claude 的“思考帽”

Claude 4.6 这里有个很关键的默认行为：如果你没特别说，它会自己决定要戴多厚的思考帽。

🎩 adaptive

像 Claude 自己看题目难不难，再决定要不要认真想久一点。不是死板固定。

/think:<level>

像你在现场跟它说：“这一题你多想一点”或者“别想太久”。

agents.defaults.models["anthropic/<model>"].params.thinking

像在后台总账本里给这位 Claude 老师预先设好思考档位，之后每次来都先按这个习惯工作。

🎈 这一段别误会

它不是在介绍一个新模型，而是在说同一个模型要不要多想、少想、自己判断着想。

第四步

⚡ /fast 不是换脑子，而是让 Claude 换跑法

这一段最容易被误读成“又一种新模型”。其实不是。它更像给同一个 Claude 按下加速键。

/fast on

像跟前台说：“这次先给我走快车道。”官方把它映射到 service_tier: "auto"。

/fast off

像说：“别抢快车道了，走普通排队线。”也就是 service_tier: "standard_only"。

🚧 这几个限制要记住

它只对 API key 路线生效。setup-token、OAuth 不吃这套。如果你把 Anthropic 请求又转发到别的代理门口，OpenClaw 也不会强行替你塞这个加速标签。

params: { fastMode: true }

像给某个模型默认贴上“优先走快一点”的小纸条。

第五步

🗂 Prompt caching 像“把刚用过的草稿纸先留在桌上”

这一段讲的是缓存，不是记忆永生。别把它想得太玄。

none

不用缓存。像每次都把桌面清空重来。

short

留 5 分钟。像刚才写过的草稿先别扔，也许马上还要接着看。

long

留 1 小时。像把草稿夹进文件夹里，今天这一阵子都先别丢。

📌 这段最关键的话

Anthropic API key 路线默认就会给你一个 short。也就是说，OpenClaw 默认会先帮你把刚写过的草稿留 5 分钟。

第六步

📜 1M context 像“超长卷轴模式”，但不是每张票都能开

官方这里很认真地提醒你：这不是普通设置，而是测试中、带门槛的超长上下文模式。

📏

params.context1m: true

像明确告诉系统：“这次我要展开超长卷轴。”不写就不会自动开。

🚫

为什么会报 429

像你想租超大会议室，但手上的票不够资格。官方明确说，没开通额外长上下文权限就会被拒。

⚠️

OAuth 票更受限

官方还专门说了：OAuth / 订阅票目前不会走这个 1M beta 头。也就是这条票路本身就不支持这扇门。

🎈

一句话

1M context 是超长卷轴，不是默认纸张；先确认你的凭证够不够格。

第七步

🖥 Claude CLI 这条路，像“把本地 Claude 柜台接到 OpenClaw 总机”

这部分很重要，因为它根本不是 Anthropic API。它是另外一条本地柜台路线。

1. OpenClaw 启动本地 claude

像请家里已经装好的 Claude 柜台开张。

2. 第一次对话会建一个 session

像第一次领一张会话号牌。

3. 后续对话会继续沿用这张号牌

像你下次回来，不重新排队，而是拿着旧号牌继续办业务。

📌 适合什么场景

更适合一台个人 Gateway 主机，已经装好并登录过 Claude CLI 的情况。它不是给多人共享计费场景准备的。

第八步

🎟 setup-token 路线，像“先去 Claude Code 机器上打一张票，再拿回来刷”

这条路线不是在 Anthropic Console 里拿钥匙，而是从 Claude Code CLI 那边拿票。

claude setup-token

像去发票机前打一张新的订阅票。这个动作可以在任何机器上做。

openclaw models auth setup-token --provider anthropic

像你站在 Gateway 主机前，直接把票塞进 Anthropic 的票夹里。

openclaw models auth paste-token --provider anthropic

像票是在别处打出来的，你现在把它抄回来，再贴到 Gateway 主机上。

⚠️ 官方真正想提醒什么

setup-token 还支持，但它不是最稳、最适合所有情况的主路。遇到失效、刷新失败之类的问题，也优先想到重新拿票。

最后总结

🎈 把 Anthropic 这页压成一句最好记的话

先选进门方式：API key、setup-token，还是 Claude CLI。选完以后，再去决定 Claude 要戴多厚的思考帽、要不要走快车道、要不要把草稿纸多留一会儿、要不要展开超长卷轴。

如果你下一步最怕“票失效、钥匙失踪、为什么又 429”，接着看网关排障页会最顺。

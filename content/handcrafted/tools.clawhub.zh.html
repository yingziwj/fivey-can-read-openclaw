先讲人话版

🏪 ClawHub 像 OpenClaw 的“技能小卖部 + 插件货架”

如果你觉得 OpenClaw 像一个会做事的小助手，那 ClawHub 就像它去补货的地方。你需要一项新本领，不一定要自己从零写，很多时候先去货架上找一盒别人已经做好的 skill 或 plugin 就够了。

所以这页真正讲的不是架构，而是“去哪里找、怎么拿回来、什么时候该用 `openclaw`，什么时候该用单独的 `clawhub` CLI”。

适合想装现成 skills / plugins，或者准备自己发布的人 这页最重要的是分清：平时拿货用 openclaw，上架和仓库管理再用 clawhub 查看官方原文

第一站

🛒 先记最重要的一句：你平时拿货，不一定要先学仓库后台

官方一开头就把两套命令分开了，这点特别对。因为大多数人只是来“找东西、装东西、更新东西”，不是马上要自己开店当仓库管理员。

🧰

openclaw 命令

像普通顾客去货架上找货、拿货、补货。平时搜索、安装、更新，优先用它。

🏬

clawhub CLI

像仓库后台和店主工具。认证、发布、删除、撤回、同步，这些更像“经营货架”的动作。

📦

货架上有什么

主要是 skills 和 plugins。你可以把它们想成“做事说明书”和“扩展能力包”。

⭐

先记一句

拿货用 openclaw，开店和管仓库再用 clawhub。

第二站

🔍 搜索、安装、更新，翻成人话就是“找货、抱回家、换新版”

这一段的命令其实特别生活化，你只要把它们想成逛小卖部就好。

openclaw skills search "calendar"

像站在货架前问店员：“有没有和日历相关的本领？”

openclaw skills install <skill-slug>

像把那盒技能真正抱回你的工作区，不是云端收藏，而是放到你桌边随时可用。

openclaw skills update --all

像检查你桌上这些技能说明书是不是已经出了新版，有的话一起换新。

openclaw plugins install clawhub:<package>

像你不是拿一张小抄，而是整箱搬回一个更大的扩展包。

第三站

📁 “装到 active workspace” 到底是什么意思

官方特别强调这一点，是因为很多人装完 skill 后会问：它到底装去哪儿了？为什么新会话里才看见？

🏠 active workspace

像你当前正在用的那张桌子。openclaw skills install 会把 skill 直接放到这张桌子的 skills/ 目录里。

📚 为什么新会话才捡起来

像老师重新发课本时，学生才会拿到最新版。已经在跑的旧会话不一定立刻重读这盒新技能。

🥇 工作区优先级更高

如果你本来在别处还有同名 skill，工作区里的版本通常更响亮，像离你最近的那份班规先算数。

🎈 这一节一句话

ClawHub 不是“远远点个赞”，而是真的把东西搬进你现在这张工作桌上。

第四站

🏗 ClawHub 后台在做什么，可以想成“仓库收货、贴标签、排货架”

官方的 How it works 一共 4 步。翻成人话，就是一次完整的仓库入库流程。

1. 发布 bundle

像作者把一盒技能或插件送到仓库门口。

2. 解析 metadata

像仓库管理员打开盒子外的标签，确认名字、版本、简介、要求是什么。

3. 建索引

像把这盒东西放进可搜索的货架目录里，以后别人按关键词、标签、用途就能找到它。

4. 用户浏览和安装

像顾客来逛店、挑货、抱回家。整套流程没有神秘魔法，就是把“别人做好的能力”变成“你能搜到并装上的能力”。

第五站

🧾 为什么官方一直提 metadata、版本、星标、下载量

因为货架不是杂物堆。要让人敢装、会选、看得懂，仓库就得把“这是什么、有没有人用、最近有没有改”这些线索摆清楚。

🏷 metadata

像盒子外的品名贴纸。没有它，别人一眼看不出这盒东西适合干嘛。

🔢 版本

像每次升级都换一张新批次编号。这样你能知道自己装的是哪一版，也能回头看历史。

⭐ stars / downloads

像货架旁边的“很多人买过”和“大家觉得不错”小贴纸，帮你快速判断热度和口碑。

🧠 embeddings search 是什么

官方说搜索不只是关键词匹配，更像店员会猜你想找哪类东西，所以你用自然语言搜，也能搜出相关货品。

第六站

🛡 开放货架一定会讲安全和 moderation

这段不能因为讲故事就讲轻。因为公开货架的好处是热闹，坏处是难免有人往里面塞奇怪东西。

🚪 谁都能上传，但不是完全没门槛

官方要求 GitHub 账号至少一周大，像至少先确认你不是刚冲进来的小号乱丢盒子。

🚨 举报机制

用户可以举报不安全或乱来的 skill。举报理由必须写，不能只是随手乱按。

🙈 超过 3 个独立举报会先隐藏

像货架上某盒东西连续被多个人指出有问题，店员会先把它撤到柜台后面，不让更多人直接拿走。

🎈 一句话

ClawHub 是开放货架，但不是没人看着的自助仓库。

第七站

🧑‍🏫 如果你只是普通使用者，最实用的路线其实很短

官方其实也替新手总结好了，只是说得没这么白。最短路径就四步。

1. 搜

先用自然语言找你要的能力。别先冲去研究仓库后台。

2. 装

找到合适的，就把 skill 或 plugin 装进当前工作区。

3. 开新会话

像让新来的技能真正发到手里，别还在旧课本里翻。

4. 想发布再去装 clawhub CLI

发布、同步、管理认证这类“店主动作”，到那时再学一点都不晚。

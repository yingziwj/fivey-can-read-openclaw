先讲人话版

Codex Computer Use 像“让 Codex 在被允许时看屏幕、点按钮”

这页讲的是一个 Codex 原生 MCP 插件。OpenClaw 不会自己偷偷控制桌面，也不会绕过 Codex 的权限；它只是帮 Codex 模式准备好能接入 Computer Use 的配置。

对小朋友讲，就是：不是 OpenClaw 自己伸手去点电脑，而是经过 Codex 和系统许可后，才把“看和点”的能力交给合适的工具。

适合理解 Codex 桌面控制边界的人 重点是 OpenClaw、Codex、MCP 和权限边界 查看官方原文

第一站

先划清三条线：谁准备、谁授权、谁执行

OpenClaw 的 codex 插件准备配置；Codex 负责实际使用 Computer Use；桌面系统和用户权限决定能不能看屏幕、点按钮。

OpenClaw

像把教室门牌和工具清单准备好。

Codex

像真正拿着工具做事的学生。

Permissions

像老师和门禁，没许可就不能随便看屏幕、点按钮。

第二站

它不是 Peekaboo，也不是 iOS 节点

官方特别把几个名字分开，是为了避免误会。OpenClaw.app 的 Peekaboo 集成、iOS app 节点、上游 cua-driver mcp，都不是同一个东西。

C

Codex Computer Use

Codex 原生 MCP 插件，重点是 Codex 模式下的桌面控制能力。

P

Peekaboo

macOS app 的另一条集成路线，权限和桥接方式不同。

I

iOS app

作为 OpenClaw node 暴露移动端能力，不是桌面控制后端。

M

Direct MCP

也可以通过 OpenClaw 的 MCP registry 接上游驱动，但那是另一种配置。

最后记住

Computer Use 最重要的不是“能点”，而是“在正确边界内能点”

这页真正要保护的是权限边界：配置可以提前准备，实际桌面动作仍要跟着 Codex、MCP server 和系统许可走。

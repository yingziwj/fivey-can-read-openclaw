先讲人话版

📘 CLI Setup Reference 像“向导老师的总答案本”

如果上一页是“怎么脚本化”，这页就是“这位向导老师到底每一栏会问什么、会填什么、会写到哪”。

它不是教你一条命令，而是把 openclaw onboard 的整套流程拆开给你看。

适合想彻底看懂 onboarding 每一步的人 这页最重要的是记住：它是总参考，覆盖本机和远端两条路 查看官方原文

总说明

🧭 先分清两条路：本机装房子，还是远端去接房子

官方开头就把 local mode 和 remote mode 拆开了。一个负责真的搭房子，一个负责只去连别人的房子。

🏡

Local mode

像在自己家里把房子搭起来，钥匙、房间、门牌都要自己准备。

🛰

Remote mode

像你这边只拿遥控器去连别处的总机房，不会替对方改房子。

🔁

重新跑向导

再跑一次不会自动把东西全删掉，除非你明确选择 Reset。

📌

一句话

本机是造房子，远端是连房子，别把两件事混在一起。

本机流程

🏠 本机模式就像“先修房，再摆门，再请值班员，再验收”

官方把本机流程拆成几个步骤，每一步都是实际会发生的事。

先看旧配置在不在

像开门前先看看屋里已经有没有你之前留过的东西。

模型和认证

像先挑脑子，再挑钥匙。到底用谁的模型、谁的门卡，都在这一轮定。

Workspace

像给小工人腾一个家，顺手把 bootstrap 文件也种下去。

Gateway

像把总机房的门牌号、门锁、朝向和外露方式都定好。

Channels

像把 Telegram、WhatsApp、Discord、Google Chat 这些门一扇一扇接上。

Daemon install

像请一个后台值班员，开机后自己上岗。

Health check

像装完房子后开灯试一遍，确认没有哪盏灯不亮。

Skills setup

像把小工人常用的工具包也一起摆好。

命令翻译

🛠 openclaw onboard 的每一步，到底在干什么？

这页最值钱的地方，就是把向导老师每一步问什么都列清楚了。

🧾

--reset

像告诉老师：如果旧房子有点乱，我愿意先收拾掉一部分再重来。

🪟

--gateway-token-ref-env

像说：门卡不要我手写，去环境变量这个抽屉里找。

📱

WhatsApp / Telegram / Discord

像不同的门都要装，老师会按顺序问你每一扇门的钥匙。

🧰

LaunchAgent / systemd

像不同操作系统里的后台值班员制服不一样，但工作本质一样。

认证和模型

🔑 这一大段，其实是在回答“钥匙怎么放，脑子怎么选”

官方把可选项写得很细，但核心还是两件事：选哪把门卡，选哪颗脑子。

API key / OAuth / setup token

像三种不同的入场方式：直接拿钥匙、走正式流程、或者拿订阅票。

Claude CLI reuse / OpenAI Code OAuth

像把你别处已经办好的门卡，搬到 OpenClaw 这边继续用。

Custom provider

像你自己开一间柜台，地址、牌子、兼容风格都能自己定。

Ref mode

像不把钥匙写死在纸上，而是写“去哪个抽屉拿”。

Remote mode

🛰 远端模式像“我只去连别人的家，不替别人搬家”

官方很强调这一点。远端模式只是在这台机器上改配置，让它连别处的 Gateway，不会去改对方那台机器本身。

🌍

只连，不改

像你带着遥控器去别人家用，不会顺手给人重新装修。

🔐

远端通常要 token

只要门不是在自家内廊，门卡就很难省。

🚪

Loopback-only 时

官方会提醒你用 SSH 隧道或 tailnet，就像别硬从窗户爬过去。

📌

一句话

远端模式只负责连线，不负责替远端打地基。

最后总结

🎈 把这页压成一句最短的话

CLI Setup Reference 是向导老师的总答案本：本机怎么装、远端怎么连、钥匙怎么存、模型怎么选、门怎么开，所有步骤都在这儿。

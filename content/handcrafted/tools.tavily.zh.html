🔎 Tavily 像一个“专门替 AI 找资料和拆网页的侦察员”

普通网页搜索像你把问题丢给搜索引擎，然后自己一条条翻结果。Tavily 更像一个已经知道 AI 需要什么的侦察员：它不只帮你找，还会尽量把结果整理成更适合模型吃的样子。

所以这页真正讲的是：你是要它“帮你找”，还是“帮你把某几个页面拆开抄重点”。前者偏搜索，后者偏抽取。

适合准备给 OpenClaw 配网页搜索，或者想要更强网页抽取的人 这页最重要的是分清：web_search、tavily_search、tavily_extract 不是一回事 查看官方原文

第一站

官方一开头就把这件事说清楚了。别跳，因为这会直接决定你后面怎么配、怎么用。

当 web_search 提供商

像把 Tavily 安排成 OpenClaw 默认的“上网找资料同学”。以后模型要搜网页时，就优先叫它。

🧰

当显式工具

也可以直接点名用 tavily_search 和 tavily_extract。这像不是让侦察员自由发挥，而是明确下达不同任务。

⭐

先记一句

一个是“默认搜索员”，一个是“手动点名执行专门动作”。

🧠

为什么 Tavily 特别适合 AI

因为它返回的结果本来就更偏向让模型读，不只是扔你一堆蓝色链接。

第二站

🔑 配置那段 JSON，翻成人话就是“给侦察员发工牌，再把总搜索台指向它”

配置块看起来像参数堆，但真正主线只有两件事：先把 Tavily 插件开起来，再告诉网页搜索以后优先找它。

plugins.entries.tavily.enabled: true

像先让 Tavily 这个侦察员正式上岗。

apiKey: \"tvly-...\"

像给它发工牌。没有这个 key，它就算站在门口也进不了系统。

baseUrl: \"https://api.tavily.com\"

像告诉它要去哪个服务台报到。大多数人直接用默认官方地址就够了。

tools.web.search.provider: \"tavily\"

像在总搜索台挂牌：“以后默认网页搜索，先叫 Tavily 出动。”

第三站

🔎 web_search、tavily_search、tavily_extract 三兄弟各干嘛

这一页如果只带走一个东西，我希望你带走这个分工表。因为绝大多数困惑都卡在“到底该叫哪一个”。

⚡

web_search

像一句“帮我搜一下”。快速、通用、选项少。你只想查资料，不想微操的时候，用它。

🎛

tavily_search

像对侦察员说：“给我搜，但这次搜深一点、只看新闻、顺手给我一段总结。”你要更多旋钮时，用它。

📄

tavily_extract

像不是去满大街找，而是已经拿到几个网址，要求侦察员回去把这些页面拆开、抄重点。

⭐

一句话判断

想“找”就搜，想“拆”就 extract。

第四站

🎚 tavily_search 的参数，像你对侦察员下的详细口令

这里别死背表格。你只要把每个参数都想成一句命令口吻，就很好记。

query

像你真正要问的问题本身。别太长，别绕弯。

search_depth: basic | advanced

basic 像“先快速扫一圈”，advanced 像“别急，给我翻深一点、找准一点”。后者更慢，但更像认真查档案。

topic: general | news | finance

像告诉侦察员这次是一般资料、新闻快讯，还是金融情报。不同场景，找的路数不一样。

include_answer

像让它别只带回链接，还顺手给你念一段“我先帮你看完的摘要”。

第五站

📄 tavily_extract 像“拿着指定网址回去抄重点”

这是 Tavily 和普通搜索很不一样的地方。你不是让它满街找，而是拿着明确门牌号，让它回去拆内容。

urls

像直接把要查的门牌号交给它。一次最多 20 个，别一口气塞太多。

query + chunks_per_source

像不是要它整本书都抄回来，而是说：“按这个问题去挑最相关的几段给我。”

extract_depth: basic | advanced

basic 像普通抄书，advanced 更像连那些 JavaScript 动态页面、复杂表格也尽量帮你拆出来。

include_images

像除了文字，也把页面里的图片线索顺手带回来。

第六站

🧠 官方那些 tips，其实都是“先走轻一点，不够再加重”

这套思路很实用。别一开始就把每个旋钮拧到最重。

🚶 先用 basic

像先普通走一圈。够用就别一上来就开最重模式。

🔁 不够再切 advanced

像发现这栋楼结构太复杂，普通巡查看不明白，再派更细的侦察。

✂️ 用 query 限定目标

像告诉它“别整篇都搬，帮我挑和这个问题最相关的几段”。这样比抱一堆原文回来更省脑子。

🎈 一句话

先轻搜、再深搜；先少搬、再多搬。

最后总结

🎈 把 Tavily 这页压成一句最好记的话

Tavily 就是一个既会帮 OpenClaw 找资料、又会帮它拆网页重点的侦察员。想快搜就用 web_search，想细控就用 tavily_search，想拆指定页面就用 tavily_extract。

如果你下一页只想继续看一页，我建议接着看通用的 /tools/web 或别的搜索提供商页。这样你会更容易比较“默认搜一下”和“专门叫 Tavily 出手”到底差在哪。

先讲人话版

ClawHub 像一间给 OpenClaw 补本领的公开仓库

OpenClaw 自己很会做事，但它不可能天生会所有事情。ClawHub 就像一排贴好标签的货架，里面放着别人整理好的 skills 和 plugins。

这页是 ClawHub 的总览：怎么搜索、怎么安装、怎么看版本、什么时候需要登录，和普通使用者、发布者分别该走哪条路。

适合第一次认识 ClawHub 的读者 重点是：找、装、看清来源、再决定要不要发布 查看官方原文

第一站

普通使用者先学“拿货”，不用先学“开仓库”

ClawHub 最常见的用法很短：搜一个能力，看看它像不像你要的，再把它装进当前 OpenClaw 工作区。

openclaw skills search "calendar"

像问货架管理员：“有没有会处理日历的小说明书？”

openclaw skills install <skill>

像把选中的说明书放到自己桌上，不只是远远收藏。

openclaw plugins install clawhub:<package>

像抱回一盒更大的能力零件，让 OpenClaw 多一种会做事的手。

第二站

货架不是杂物堆，每一盒都要有标签

ClawHub 会记录版本、标签、更新记录、文件、下载量、星标和安全扫描摘要。对小朋友讲，就是每盒东西外面都贴着“我是谁、谁做的、有没有人用过”。

1

版本

像玩具盒上的批次号，出问题时能知道是哪一版。

2

标签

像“搜索”“日历”“语音”这样的分类贴纸，帮你更快找到。

3

安全摘要

像入门前的检查单，提醒你这盒东西有没有值得小心的地方。

4

来源记录

像收据。以后更新时，OpenClaw 才知道要回哪条货架找新版。

最后记住

ClawHub 让“别人做好的能力”变成“你能放心找到的能力”

普通读者先掌握搜索和安装就够了。等你想把自己的 skill 或 plugin 放上货架，再去学习登录、发布、owner 和版本这些店主动作。

先讲人话版

📍 Location Command 这页，其实是在讲“节点能不能把它现在的位置报出来，以及报到多精确”

这不是默认全开的能力，而是要看设备设置、系统授权、以及是否允许精确定位。

节点定位能力重点是 off / whileUsing / preciseEnabled查看官方原文

第一站

🧭 官方没用简单开关，而是用“模式选择器”

因为位置权限本来就不是单一开或关。

⛔

off

完全不报位置，像把定位芯片盖住。

🚶

whileUsing

只在你正在使用 app 时给位置，比较像“当面借用一下地图”。

🎯

preciseEnabled

这决定给的是精确坐标还是比较模糊的大概区域。

📱

OS 最终拍板

应用能请求，但系统设置才是真正最后盖章的人。

第二站

🪪 UI 里的选择只是“我想申请什么级别”，不等于系统一定答应

所以官方才会强调：选择器驱动请求，实际授予还是看操作系统。

iOS / macOS

系统可能给你 While Using，也可能有更高一档 Always，但最终决定权不在 OpenClaw 手里。

Android

当前更偏前台定位，说明这边管得更紧。

location.get

这就是实际问设备“你现在在哪儿”的命令入口。

🎈 最后记一句

这页的重点不是地图本身，而是权限层级和精度层级。

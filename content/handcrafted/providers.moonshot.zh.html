先讲人话版

🌙 Moonshot 这页，其实是在提醒你：Moonshot 和 Kimi Coding 是两扇不同的门

很多人一看到都是 Kimi，就以为同一把钥匙全能开。官方专门强调不是这样。Moonshot API 和 Kimi Coding 是两套门、两把钥匙、两种模型前缀。

适合想接 Kimi 系列模型的人重点是 Moonshot 和 Kimi Coding 分家、thinking 开关查看官方原文

第一步

🚪 先分清两扇门：moonshot/... 和 kimi-coding/...

这是这页最重要的一件事。别混。

🌙 Moonshot API

像 Moonshot 自家的主门，模型前缀写成 moonshot/...。

💻 Kimi Coding

像给编程场景准备的另一扇门，前缀写成 kimi-coding/...。

⚠️ 为什么官方重复强调

因为这两扇门的钥匙不通用，入口地址也不一样。拿错钥匙去敲门，只会撞门。

第二步

🧠 Kimi 这些型号，主要是在区分“普通答题”和“思考版”

kimi-k2.5、kimi-k2-thinking 这一串名字，看起来像很多新脑子，其实有一条主轴很清楚。

💬

kimi-k2.5

像普通答题版老师。

🤔

kimi-k2-thinking

像会多想几步的思考版老师。

⚡

turbo

像同一路老师里的加速档。

🎈

一句话

大方向还是普通版和思考版，再加一个快一点的档位。

第三步

🪄 native thinking 像“给 Kimi 戴上或摘掉思考帽”

这页最后那段最值得翻成人话，因为它其实很好懂。

thinking: { type: "enabled" }

像明确说：“这轮把思考帽戴上。”

thinking: { type: "disabled" }

像说：“别拐弯了，直接答。”

/think off

像会话现场把思考帽摘掉。

工具选择为什么会被自动改

官方说当 thinking 开着时，某些 tool_choice 不兼容，所以 OpenClaw 会像现场协调员一样，自动把不合适的工具选择改回能兼容的 auto。

最后总结

🎈 把 Moonshot 这页压成一句最好记的话

先分清 Moonshot 和 Kimi Coding 是两扇不同的门，再决定这次请的是普通版 Kimi，还是戴着思考帽的 Kimi。

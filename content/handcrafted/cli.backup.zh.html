先讲人话版

🧳 openclaw backup 像“把家里的重要东西装进一个防丢的大包”

这页讲的是备份。不是把房子搬走，而是先把最重要的东西收进包里：配置、钥匙、会话、聊天记录，还有有时候连整个工作区也一起装进去。

它像出门前把证件、钥匙和相册都先放好。真出事了，你还能把家找回来。

适合想先存一份“我现在这套 OpenClaw 长什么样”的人 这页最重要的是记住：backup 是“先留底”，不是“换新机器” 查看官方原文

第一站

🧺 它会装进去哪些东西？先把柜子里的几样宝贝一口气捆好

官方一开始就说得很清楚：这不是只抄一个配置文件，而是把 OpenClaw 的几类重要家当一起打包。

🧠

state

像家里的总记忆本，很多运行中的痕迹都在这里。

🧾

config

像房间墙上的总规则，告诉系统怎么住、怎么跑。

🔐

credentials

像门卡和小钥匙，丢了就进不了门。

💬

sessions

像聊天抽屉里的小故事，很多现场上下文都在里面。

第二站

🌳 `workspace` 也可以一起装，但你也能说“先别装院子”

默认情况下，备份会把工作区也找出来一起带走。这样最完整。可你要是只想先快一点，就能让它少装一点。

openclaw backup create

像把整套家当先装箱。默认会把状态、配置、凭据、会话和工作区都考虑进去。

--no-include-workspace

像对搬家师傅说：“院子先别搬，先把屋里这些最重要的东西打包。”

--only-config

像只带走一张最关键的地图，别的先不碰。

一句话

想最全就一起装，想快一点就少装，想最轻就只拿配置。

第三站

📦 `--output`、`--dry-run`、`--verify`，是在决定“包放哪、先不先开箱、装得对不对”

命令参数看起来多，其实都在管搬箱子的流程。

📍

--output ~/Backups

像告诉搬家车：“打包好的包，先放到这个仓库去。”

🪞

--dry-run

像先把动作演一遍，不真的把箱子封起来。

🧪

--verify

像打包后再让验货员摸一遍，看看盒子里是不是少了东西。

📌

一句话

output 管放哪，dry-run 管先演练，verify 管查漏补缺。

第四站

🗂️ 为什么它会带一个 `manifest.json`？因为箱子里要放清单

官方特别提到清单文件。这个设计很像收纳盒外面贴一张标签纸，里面放了什么，一眼能看懂。

清单本

manifest.json 像箱盖上的总目录，告诉你里面每样东西原来从哪来。

不乱装

它会先把路径整理好，避免把同一个东西重复装两次。

不乱放

输出目录如果放进原来的家当堆里，它会拒绝，怕你把备份卷进备份里。

像贴标签

备份不是黑箱。它是一个贴好标签的箱子，方便以后真要找回东西时不迷路。

第五站

⚠️ 配置坏了也想先救命？它会先帮你拿到“能救多少救多少”

官方这段很实用。它的意思是：备份这件事本来就是为了救急，所以不能太脆。

🚑

正常模式

如果配置还健康，就按原计划把该备的都备上。

🧯

坏配置时的退路

如果 workspace 解析要依赖的配置已经坏了，backup create 会直接说不行。

🧺

--no-include-workspace

像把最麻烦的院子先跳过去，先保住屋里那些救命的东西。

📌

一句话

备份这件事，宁可少一点，也别在救命时卡死。

第六站

🧷 它的速度和大小，主要看房子有多大、路有多远、箱子会不会挤

官方没有给硬上限。这个意思很朴素：文件多、工作区大、磁盘慢，打包自然就慢一些。

空间

箱子要先有地方放，不然你连封口都封不上。

时间

房子越大，搬完一圈越花时间。

文件系统

有些地方能挂钩、有些地方只能整包拷走，速度和方式就不一样。

最轻的包

如果你只想要最小的一包，就用 --only-config。像只拎最重要的地址本。

最后总结

🎈 把 openclaw backup 压成一句话

openclaw backup 就像给 OpenClaw 打一个贴着清单的大包：把状态、配置、钥匙、会话和工作区收好，存到安全地方，真出问题时还能把家找回来。

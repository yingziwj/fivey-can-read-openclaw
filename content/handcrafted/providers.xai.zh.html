先讲人话版

🚀 xAI 这页其实是在说：一把 xAI 钥匙，不只开 Grok 聊天门，还能开搜索门和远程代码箱

官方这页最特别的地方，是它不只是模型 provider 说明。它还顺手告诉你：同一把 XAI_API_KEY 还能给 xAI 的搜索和远程代码执行能力供电。

适合想用 Grok / xAI 全家桶的人重点是同一把 key 复用、搜索和远程代码箱查看官方原文

第一步

🔑 主路还是拿 XAI_API_KEY

这里没绕路。先在 xAI 控制台办钥匙，再让 OpenClaw onboarding 收下它。

🛠

openclaw onboard --auth-choice xai-api-key

像在向导里明确说：“我要走 Grok 这扇门。”

🧠

xai/grok-4

像先请 Grok 家这位大老师默认坐班。

🎈

一句话

xAI 入口先从 API key 开始，没有 OAuth 小路可走。

第二步

🧰 同一把 key 还能供电给搜索和远程代码箱

这是这页最值得记住的地方，不然很容易把 xAI 误会成“只是又一家聊天模型”。

🔎 web_search / x_search

像同一把钥匙还能打开 xAI 家的侦察员工具间。

🧪 code_execution

像还能打开远程代码实验箱，但注意这是 xAI 远端沙箱，不是你本地的 exec 工具。

📌 复用逻辑

官方甚至说，如果你把 xAI key 存在插件搜索配置里，模型 provider 也会拿它当候补。这说明它们在尽量复用同一把钥匙。

第三步

🚧 已知限制，像“这扇门现在还没装所有通行方式”

官方这里也很坦白。

目前只有 API key 路线，没有 OAuth / 设备码。还有一个实验性的 grok-4.20-multi-agent-experimental-beta-0304 也不走普通 provider 路。所以别以为所有 Grok 名字都能直接塞进同一条管道。

最后总结

🎈 把 xAI 这页压成一句最好记的话

xAI 不只是聊天模型入口。同一把 XAI_API_KEY 还能顺手给搜索和远程代码箱供电，但它目前仍是纯 API key 路线。

先讲人话版

🪪 Plugin Manifest，就是插件的身份证和说明卡

这页讲的是插件装上去之前，系统最先要看的那张卡。

名字、版本、入口、配置、能力，先写好，OpenClaw 才知道这盒东西能不能放进来。

适合先看 manifest 长什么样 重点是字段、类型、默认值和入口 查看官方原文

第一站

📇 先把卡片写清楚，系统才认得你

manifest 不是装饰，它是让 OpenClaw 读懂“你是谁、能做什么、从哪开始读你”的基础。

🏷

id / name

像名字牌，别人先知道你是谁。

🚪

entry

像正门门牌，告诉系统从哪进。

🧰

capabilities

像技能标签，告诉系统你会什么。

第二站

🛠 配置像在做什么

官方列字段的目的，是让你先把身份证写对，再谈插件能干活。

`defaults`

像先把默认衣服放好，省得每次都重新穿。

`optional`

像一张“先放柜子里”的贴纸。

`schema`

像告诉系统这张卡片哪些地方不能乱填。

最后记住

🎈 Manifest 是“先写卡，再进门”

这页把卡片写明白，后面的注册、安装、运行才不会出错。

先讲人话版

🪵 Bedrock Mantle 像 AWS 里的一扇模型大门

它走的是 OpenAI 兼容接口，但钥匙来自 AWS 身份链或者显式 bearer token。你可以把它理解成“Bedrock 里面多了一层 Mantle 门厅”。

这页最重要的是：先把 AWS 认证链打通，再让 OpenClaw 自动发现模型。

适合 AWS 里跑模型的人 重点是 bearer token、region、自动发现 查看官方原文

第一步

🔑 先决定你是用现成门票，还是走 AWS 身份链

官方把两条路都写出来了，省得你卡在认证上。

🎟

显式 bearer token

AWS_BEARER_TOKEN_BEDROCK 直接给一张现成门票。

🪪

IAM credential chain

让 AWS 自己从你现成的身份链里把票变出来。

🌍

Region

像选去哪一栋 AWS 楼层，默认 `us-east-1`。

第二步

🔎 自动发现模型，像先去大厅查目录

只要认证通了，OpenClaw 会去 /v1/models 看今天有哪些模型可用。

自动发现

像前台直接去大厅名单板抄名单。

缓存

发现结果会缓存一小时，避免每次都重跑。

适用模型

GPT-OSS、Qwen、Kimi、GLM 这类托管模型都可能在这里出现。

第三步

📦 手写配置时，门牌就得自己填清楚

如果你不走自动发现，也能手写 baseUrl 和模型列表。

amazon-bedrock-mantle

像这扇门在 OpenClaw 里的正式门牌。

/v1/chat/completions

像统一的对话窗口。

AWS 区域兼容

支持的 region 很多，但要跟你的身份和门牌对应上。

最后总结

🎈 把 Bedrock Mantle 这页压成一句最好记的话

Bedrock Mantle 是 AWS 里的模型门：先把身份链打通，再让 OpenClaw 自动去大厅找模型，必要时再自己手写门牌。

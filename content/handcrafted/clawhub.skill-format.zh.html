先讲人话版

Skill format 就是在教你“一个技能盒子该怎么装”

ClawHub 不能只看到一堆文件就猜出它们是什么。一个 skill 要像整齐的小资料夹：有 SKILL.md，有名字，有简介，必要时还要声明它需要什么运行环境。

这页最重要的不是背字段，而是明白：清楚的格式能让别人看懂、能让搜索找到、也能让安全检查更靠谱。

适合要写 skill 并发布到 ClawHub 的人 重点是 SKILL.md、frontmatter 和运行需求 查看官方原文

第一站

一个 skill 先是一整个文件夹，不只是一句话

文件夹里最关键的是 SKILL.md。它像封面和说明书，告诉 OpenClaw 这个 skill 叫什么、会做什么、该什么时候用。

文件夹

像一个透明资料袋，里面可以放说明、脚本、示例和资源。

SKILL.md

像资料袋封面。没有它，别人不知道这袋东西是不是一个正式 skill。

可选支持文件

像附带的小工具和参考图，能帮助 skill 真正把事情做好。

第二站

frontmatter 像盒子顶部的小标签

YAML frontmatter 放在 SKILL.md 开头。它不是给人看的长故事，而是给 ClawHub 和机器快速读取的小标签。

N

name

这盒 skill 的名字。

D

description

一句短说明，搜索结果里常常靠它让人判断。

V

version

像第几版说明书，方便更新和回退。

R

runtime metadata

告诉系统它可能需要哪些 OpenClaw 能力或运行条件。

小心事项

格式写清楚，是在帮未来的自己

名字别太像别人的，简介别写成谜语，运行需求别藏着不说。ClawHub 越能读懂这个 skill，用户越容易放心安装。

先讲人话版

Manage plugins 像“给 OpenClaw 装零件、换零件、拆零件”

插件管理不需要先背很多理论。大多数时候就是几步：列出已有插件、搜索新插件、安装、重启 Gateway、确认能用，不需要时再卸载。

这页适合把 openclaw plugins 的日常动作串起来看，像整理一盒工具。

适合日常安装和维护 plugin 的读者 重点是 list、search、install、update、uninstall 查看官方原文

第一站

先看桌上已经有什么零件

安装新东西前，先列出现有插件。这样你知道哪些已经启用、哪些只是放着、哪些可能正在影响系统。

openclaw plugins list

像打开工具箱，看看里面有哪些零件。

--enabled

只看正在通电工作的零件。

--verbose

把标签看得更细，包括来源、版本和更多状态。

第二站

搜索和安装像“先问店员，再把盒子搬回家”

ClawHub 是主要货架之一。裸包名、ClawHub 包和本地目录都可能成为安装来源，但越是外来的东西，越要先看清来源和说明。

1

Search

先找，不急装。关键词越清楚，越容易拿到合适结果。

2

Install

把选中的 plugin 记录到 OpenClaw 的插件位置。

3

Restart

Gateway 常常要重新读配置，像机器换了零件后重开电源。

4

Verify

装完要确认真的出现、真的启用、真的能被调用。

第三站

更新和卸载都先用 --dry-run 想一想

更新像换新版零件，卸载像把零件从机器里取出来。影响越大，越适合先 dry run 看看会发生什么。

openclaw plugins update --all

像给所有插件检查新版，但要留意兼容性。

openclaw plugins uninstall <id> --dry-run

先演练拆零件，不马上真的动手。

发布到 ClawHub 或 npm

想把自己的插件分享出去，就从“使用者动作”升级成“维护者动作”。

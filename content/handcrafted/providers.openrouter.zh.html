先讲人话版

🛍 OpenRouter 像一个超大模型商场：你带一把门卡进去，里面很多柜台都能逛

这页官方内容其实很简单。OpenRouter 不是某一颗单独模型脑子，而是一个统一入口。你给它一把 API key，它帮你把请求转去不同 provider 的不同模型。

所以这里最关键的，不是学一堆参数，而是先接受这个概念：你不是直接敲某家店的门，而是在模型商场的总前台登记。

适合想一把钥匙接多家模型的人 重点是模型命名格式和统一入口思维 查看官方原文

第一步

🔑 先把 OpenRouter 门卡交给 OpenClaw

这一步没有花活，就是认证。

🛠

openclaw onboard --auth-choice apiKey --token-provider openrouter --token "$OPENROUTER_API_KEY"

像在总前台说：“我要进模型商场，这是我的门卡。”

📦

OPENROUTER_API_KEY

这张卡不是某一家模型专属钥匙，而是进总商场的统一通行证。

🏠

放哪里最稳

跟其他 provider 一样，最好放到 Gateway 长期值班时也能读到的地方。

🎈

一句话

OpenRouter 先认你的商场门卡，再帮你转去具体柜台。

第二步

🏷 模型名字为什么这么长？因为它在写“商场 / 店家 / 柜台”

官方最值得你看懂的，就是这个命名格式。

openrouter/anthropic/claude-sonnet-4-6

第一段 openrouter 像商场名字，第二段 anthropic 像进了哪家店，第三段才是柜台上具体哪位模型老师。

为什么不是直接 anthropic/...？

因为你此时不是直连 Anthropic，而是先过 OpenRouter 这个总转运站。

🧭 这页真正想告诉你的事

OpenRouter 最有价值的地方，不是“名字长”，而是你能用同一种入口去切很多 provider 和模型。

🎈 一句话

模型名越长，不是越复杂，而是在把路径写完整。

第三步

📮 Bearer token 在现场像什么？像你进商场时给门卫看的通行证

官方最后一句说 OpenRouter 底层用 Bearer token。翻成人话，就是请求每次都会带着那张门卡去前台验票。

所以你看到 env: { OPENROUTER_API_KEY: "sk-or-..." } 这种配置，不是在“背参数”，而是在往总前台的验票机里塞通行证。

最后总结

🎈 把 OpenRouter 这页压成一句最好记的话

OpenRouter 不是某一个模型，而是模型商场总前台。你交一把门卡进去，再用 openrouter/店家/模型 这种路径告诉 OpenClaw 你到底要去哪个柜台。

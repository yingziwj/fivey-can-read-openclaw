先讲人话版

🔎 Ollama Web Search 像给本地 Ollama 再开一扇“搜网页”的门

它不是模型 provider，而是 web_search provider。意思是：这扇门负责上网找资料，不负责聊天大脑。

它很适合本地 Ollama 用户，因为能复用同一台机器和同一个登录状态。

适合已经在用 Ollama 的人 重点是 `ollama signin` 和 `openclaw configure --section web` 查看官方原文

第一步

🪪 先登录 Ollama，再把它选成搜索门

这不是直接填一个 API key，而是先让 Ollama 自己知道你是谁。

1

ollama signin

像先去 Ollama 前台刷脸。

2

openclaw configure --section web

像去网页搜索房间里选门。

3

provider = ollama

像把搜索任务交给这扇门。

第二步

🏠 它会复用你已有的 Ollama host

如果你已经拿 Ollama 跑模型，它就顺手接着用同一台主机。

默认 host

127.0.0.1:11434。

自定义 host

你也可以写自己的远程 Ollama 地址。

Bearer auth

如果 host 要认证，OpenClaw 还能复用同一套 key。

第三步

📦 这页最容易混的点：它是搜索，不是模型

不要把 Ollama Web Search 跟 Ollama 模型 provider 混成一件事。

输出

标题、URL、snippet 这种结构化结果。

不用单独的搜索 key

它默认不需要 web-search 专门 API key。

实验性 endpoint

底层走 Ollama 的 experimental web search API。

最后总结

🎈 把 Ollama Web Search 这页压成一句最好记的话

这页就是给 Ollama 再开一扇搜网页的门：先登录，再选 provider，最后复用你已有的本地 host。

先讲人话版

🧵 `openclaw tasks flow` 像给后台任务再套一层“流程衣服”

它不是独立的 `flows` 主命令，而是 `openclaw tasks` 下面的子命令。

这页最重要的是：它管的是 Task Flow，不是普通单个 task。

适合想操作任务流程的人 重点是 list / show / cancel 三个动作 查看官方原文

第一站

📚 先认命令位置：它住在 tasks 下面

这页开头就把误会掐掉了。

openclaw tasks flow list

像列出所有流程。

openclaw tasks flow show <lookup>

像打开某条流程看详情。

openclaw tasks flow cancel <lookup>

像把某条流程按停。

第二站

🪢 它管理的是“多步流程”，不是单次跑腿

Task Flow 适合一串步骤接着一串步骤的事。

1

先列出

像看有哪些流程在账本里。

2

再查看

像打开某条流程的说明书。

3

必要时取消

像给流程踩刹车。

📌

和 tasks 的区别

tasks 记单个后台工作，flow 记整条流程。

第三站

📦 这页其实是在帮你找对那三条路

流程命令没那么多，别把它想复杂了。

list

像列清单。

show

像看细节。

cancel

像按停按钮。

最后总结

🎈 把 CLI Flows 这页压成一句最好记的话

`openclaw tasks flow` 不是单独的大门，而是 `tasks` 下面的流程抽屉：列、看、停，三件事就够了。

先讲人话版

Memory Wiki 像把零散记忆整理成一本会标注来源的百科书

普通 memory 像一盒便签，记着很多事实、偏好和经历。memory-wiki 插件会把这些便签整理成更稳定的知识库页面。

它不只是写漂亮段落，还会关心证据、来源、置信度、矛盾和还没弄清的问题。

适合想理解 durable memory 如何变成知识库的读者 重点是 vault、provenance、confidence 和 hybrid pattern 查看官方原文

第一站

便签和百科书不是同一种东西

memory-core 更像收集原始便签，memory-wiki 更像把便签整理成页面。一个负责记住，一个负责把记住的东西排版、归档、注明依据。

Memory

像抽屉里的小纸条，快、碎、灵活。

Wiki

像整理好的百科页，有标题、结构和来源。

Evidence

像每条结论旁边贴着“我是从哪里来的”。

第二站

好的 wiki 不只说“我知道”，还会说“我有多确定”

这个插件强调 provenance、confidence、contradictions 和 open questions。翻成人话，就是它会尽量说明来源、确定程度、冲突点和待确认事项。

P

Provenance

来源记录，像书后面的参考资料。

C

Confidence

确定程度，像“我很确定”或“还要再确认”。

!

Contradictions

发现前后矛盾时，不假装没看见。

?

Open questions

把还没搞懂的问题留下来，方便下次继续查。

最后记住

Memory Wiki 的目标不是记更多，而是把记忆整理得更可检查

适合长期使用 OpenClaw 的场景：让重要知识有页面、有来源、有置信度，而不是永远散在一堆便签里。

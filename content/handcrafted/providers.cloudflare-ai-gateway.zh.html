先讲人话版

☁️ Cloudflare AI Gateway 像“模型前面的收费站和观察塔”

这页讲的不是新模型，而是在原本 provider 前面再套一层 Cloudflare 关卡。你还是在用后面的模型，比如 Anthropic，只是中间多了一个会做统计、缓存和控制的收费站。

适合想在模型前面加 Cloudflare 关卡的人重点是双层凭证：后端模型钥匙 + Cloudflare 关卡票查看官方原文

第一步

🚧 先理解它不是模型，是前置关卡

很多人一看到 provider 名就以为又是一家模型。其实不是。它更像一条你自己搭出来的前置通道。

后面的模型

真正干活的仍然是 Anthropic 之类的模型老师。

前面的 Gateway

像你在老师办公室外面先加了一层门岗，负责记账、观察、缓存和限流。

🎈 一句话

Cloudflare AI Gateway 是前面的关卡，不是后面的脑子。

第二步

🔑 这里有两种票，别混

官方这里很容易让人看花眼，因为同一个请求里可能同时带两种凭证。

🧠

provider API key

像真正给后排模型老师看的钥匙。比如 Anthropic 的 key。

🛂

CLOUDFLARE_AI_GATEWAY_API_KEY

像通过 Cloudflare 前置关卡时出示的通行证。

🧷

cf-aig-authorization

如果你还给这道关卡自己加了认证，就得额外再贴一张 Cloudflare 内部门票。

🎈

一句话

一张票给前门，一把钥匙给后厨。

第三步

🛠 那条 onboarding 命令，像“把账户号、关卡号、通行证一次登记完”

openclaw onboard --auth-choice cloudflare-ai-gateway-api-key 不是魔法，它就是带你填 Cloudflare 这层门岗需要的信息。

而那条非交互式命令里出现的 account-id、gateway-id、API key，像在给关卡管理员写清楚“你是哪家公司、哪座收费站、拿哪张票通行”。

最后总结

🎈 把 Cloudflare AI Gateway 这页压成一句最好记的话

它是在真实模型前面再加一道 Cloudflare 关卡。别把“前门通行证”和“后面模型钥匙”混成一把。

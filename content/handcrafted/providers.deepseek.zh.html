先讲人话版

🐋 DeepSeek 这页很短，因为它其实只在教你：把钥匙放进去，然后选“普通脑”还是“思考脑”

官方这页没有太多绕路。DeepSeek 在 OpenClaw 里走的是 OpenAI 兼容路线，所以整体感觉就是：拿 API key，选默认模型，然后让 Gateway 能读到这把钥匙。

真正需要分清的，只有两个模型：deepseek-chat 和 deepseek-reasoner。

适合想快速接好 DeepSeek 的人 重点只有钥匙、默认模型、守护进程环境变量 查看官方原文

第一步

🔑 先把 DeepSeek 钥匙交给 OpenClaw

这页最核心的动作就是认证。没有钥匙，后面模型名字写得再漂亮也只是空牌子。

🛠

openclaw onboard --auth-choice deepseek-api-key

像在新手向导里直接说：“我要接 DeepSeek，请收下这把钥匙。”

⚡

非交互式命令

那串带 --non-interactive 的命令，像你不想聊天，直接把表格一次填完交上去。

📦

DEEPSEEK_API_KEY

这就是那把真正的钥匙。最好放到 Gateway 能长期读到的地方。

🧹

--skip-health --accept-risk

像跳过体检、直接签字开跑。能用，但更像自动化脚本场景，不是最稳的新手路线。

第二步

🧠 两个模型，其实就是两个脑袋档位

官方给你的选择不复杂。别被表格吓到，本质就是“快聊”和“认真想”。

deepseek/deepseek-chat

像普通聊天脑。更适合日常对话、一般回答、速度优先的工作。

deepseek/deepseek-reasoner

像戴着思考帽的脑袋。更适合需要多想几步、推一推、算一算的任务。

📏 128K context 在说什么

像这两个脑袋都自带一张挺长的桌子，能摊开比较多上下文。

🎈 最好记的话

chat 是普通模式，reasoner 是认真想模式。

第三步

🏠 守护进程环境变量，就是“值班门卫能不能摸到钥匙”

官方专门补了这一句，是因为很多人本地命令能跑，服务一挂到后台就突然找不到 key。

🪟 前台终端能看到钥匙

像你手里拿着钥匙，自己当场开门，当然没问题。

🏢 launchd / systemd 后台值班

像换成夜班门卫在看门。如果你没把钥匙交给他，他就会一脸茫然。

~/.openclaw/.env

像给值班室留一份稳定钥匙清单。官方点名推荐这种地方。

🎈 这句一定记住

你不是把钥匙给“你自己现在这个终端”，而是给“真正跑 Gateway 的那个进程”。

最后总结

🎈 把 DeepSeek 这页压成一句最好记的话

先把 DEEPSEEK_API_KEY 放稳，再决定默认用普通聊天脑还是思考脑，最后确认后台值班进程也摸得到这把钥匙。

如果你下一步要比较“哪家模型商店接入更省事”，可以接着看 OpenRouter 或 Google 那几页。

先讲人话版

🌬 Mistral 这页其实是在说：同一家门口，不只接文字老师，还接听写员和记忆贴纸机

Mistral 在 OpenClaw 里不只是普通 LLM。官方顺手把三种活都挂在这里了：文本/图片回答、音频转写、记忆向量。

适合想把 Mistral 一家多用的人重点是文本、Voxtral 转写、embeddings查看官方原文

第一步

🔑 基础接法很直：给 MISTRAL_API_KEY，默认请大老师值班

这里没太多花样，主路就是 API key。

🛠

openclaw onboard --auth-choice mistral-api-key

像在前台说：“我要接 Mistral，请收钥匙。”

⚡

--mistral-api-key "$MISTRAL_API_KEY"

像你已经把钥匙拿在手里，直接塞进去。

🧠

mistral/mistral-large-latest

像默认先请一位大老师坐班。

🎈

一句话

基础接法很短，重点在后面的多功能用法。

第二步

🎧 Voxtral 像 Mistral 家的“听写员”

官方专门给了音频转写示例，就是在告诉你：Mistral 不只会答字，还能先听音频。

tools.media.audio.models: [{ provider: "mistral", model: "voxtral-mini-latest" }] 这段配置，像把一位名叫 Voxtral 的听写员安排在门口，遇到语音消息先抄成文字，再交给后面的老师处理。

第三步

🧠 embeddings 路线，像“给记忆盒贴坐标标签”

官方最后一句提到 memory embeddings。翻成人话，就是把内容压成便于检索的小坐标。

memorySearch.provider = "mistral"

像告诉记忆管理员：“以后给记忆卡片贴标签，用 Mistral 这套贴纸机。”

mistral-embed

像这台贴纸机的默认型号。

最后总结

🎈 把 Mistral 这页压成一句最好记的话

Mistral 不只是普通文本模型入口，它还顺手提供听写员 Voxtral 和记忆贴纸机 embeddings，是一家多用的门口。

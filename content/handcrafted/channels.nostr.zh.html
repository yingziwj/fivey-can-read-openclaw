先讲人话版

🔐 Nostr 这页，其实是在教你“给 OpenClaw 一把私钥，让它去分布式世界里收加密私信”

它不像普通聊天平台那样有一个总后台。Nostr 更像很多中继站拼起来的街道网络。

所以这页最关键的三件事是：装插件、准备私钥、告诉 OpenClaw 去哪些 relay 门口等消息。

可选插件，默认不启用 重点是私钥、relay、DM 策略 查看官方原文

第一站

🧩 先把插件装上，不然 OpenClaw 根本不会这门方言

这一步很像先给机器人装“会看 Nostr 私信”的耳朵。

openclaw onboard

像新手老师带着你走流程。选到 Nostr 时，会顺手问你要不要把插件装上。

openclaw plugins install @openclaw/nostr

这句像去插件商店领现成包裹，把 Nostr 这块能力搬进来。

--link <path>

这不是联网下载，而是直接把你本地源码那份插件接进来，更像开发者走内部通道。

🔄 重启 Gateway

插件装完要重启，就像新耳朵装好后，得重新开机它才会真正开始听。

第二站

🗝️ 私钥不是装饰品，它就是“机器人在 Nostr 世界里的身份证和印章”

没有这把私钥，OpenClaw 就没法解密私信，也没法代表自己发消息。

nak key generate

这句像现场给机器人打一把新钥匙。如果你还没有 Nostr 身份，就从这里开始。

export NOSTR_PRIVATE_KEY="nsec1..."

这句像先把钥匙放进外套内袋，等 OpenClaw 开门时再摸出来用。

privateKey: "${NOSTR_PRIVATE_KEY}"

这不是再造一把钥匙，而是在配置里写“去口袋里拿”。这样比把真钥匙硬写在账本上安全。

--use-env

这句的意思很简单：钥匙继续放环境变量里，不要抄进配置文件长期保存。

第三站

📡 relay 就像一排消息中转站，你得告诉机器人去哪些站台蹲消息

Nostr 不是一座大楼，而是一堆分散站点。OpenClaw 要知道去哪几站收信。

--relay-urls "wss://..."

这句像把几个站台地址抄给机器人，告诉它“去这些地方等车”。

dmPolicy: "pairing"

默认像门卫先发配对码。陌生人先报号，过了再聊，比较稳。

allowlist / open / disabled

allowlist 是白名单小本本，open 是谁都能敲门，disabled 就是把私信门直接关掉。

allowFrom

这里放的是允许来敲门的公钥名单，像门口准入名单。只写这里的人，门卫才放行。

第四站

🖼️ profile 那段，不是聊天功能本体，而是在给机器人挂头像、贴名牌

如果你想让它在 Nostr 世界里看起来像个完整角色，就会用到这部分。

🏷️

name / displayName

像正式名字和展示名字，一个给系统认，一个给别人看。

📝

about

像胸前小卡片，告诉大家“我是干什么的”。

🖼️

picture / banner

是头像和横幅。官方还特别提醒，图链最好走 https://。

🎈

最后记一句

Nostr 这页真正的主线只有一句话：装耳朵、拿钥匙、找站台、定门卫规则。

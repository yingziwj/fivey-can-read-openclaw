先讲人话版

🪶 Feishu 这页，其实是在教你“给 OpenClaw 接一只飞书企业机器人，而且尽量别把 webhook 门暴露到外面”

它的好处很直接：收消息这边走 WebSocket 订阅，所以不像很多通道那样一上来就逼你开公网 webhook。

这页真正的核心只有三样：飞书应用、App ID / Secret、Gateway 状态检查。

适合企业飞书 / Lark 机器人 重点是 App ID、App Secret、WebSocket 订阅 查看官方原文

第一站

🤖 这条路更像“先在飞书后台领一只机器人，再把它的身份证交给 OpenClaw”

不是 token 花活，而是企业应用接入流程。

🏢

Open Platform

像企业机器人的办证大厅。

🪪

App ID / Secret

像机器人的身份证号和私密钥匙，后面全靠它们对接。

🔌

WebSocket 订阅

像给机器人开一条长连线，不必先在公网搭收件门。

🎈

一句话

Feishu 这条路主打“企业机器人 + 长连接”，不是“先开公网洞”。

第二站

🛠️ onboarding 和 channels add，都只是“把身份证录进系统”

选向导还是手工 CLI，差的是入口，不是世界观。

openclaw onboard

像让新手老师带你一步步办证、录信息、点亮通道。

openclaw channels add

像你已经熟了，直接走人工窗口，把 App ID 和 Secret 填上去。

domain: \"lark\"

如果你用的是国际版 Lark，这就像告诉系统“别去中国区大厅，去全球区大厅办证”。

🎈 收尾一句

飞书这页最重要的，不是命令，而是“你拿到的那只企业机器人身份”要填对。

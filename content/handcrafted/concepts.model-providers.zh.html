先讲人话版

🏬 Model providers 像“脑子供应商、货架规则和插头接口说明书”

这页不是在教你怎么聊天，而是在解释 OpenClaw 和不同模型供应商之间到底怎么对接、怎么轮换、怎么让插件把新货架插进来。

如果把 OpenClaw 想成一间脑子装配厂，这页就是供应链说明书。

适合想弄懂 provider 体系和扩展方式的人 这页最重要的是记住：provider 不只是模型名，还包含认证、目录、运行时行为 查看官方原文

快速规则

🏷 最先要记住的，是 provider/model 这种门牌格式

官方开头的 quick rules 里，这条最关键。因为后面所有 allowlist、切换、fallback 都靠它认路。

🏷

provider/model

像完整货号。前面是哪家供货商，后面是哪颗脑子。

📚

agents.defaults.models

像这间工厂准许上架的脑子目录。写了它，就相当于开了 allowlist。

🪄

CLI helpers

像导购老师：openclaw onboard、models list、models set 都是在帮你别自己手抠总账本。

🧩

Provider plugins

像可插拔的新货架。插件不只带模型名单，还能带运行时行为。

插件管什么

🧰 Provider 插件不只带目录，它更像“供货商整套运营手册”

官方列了一长串 hook 和能力名。别硬背，把它们想成供应商自己接手的那一套细活。

Onboarding / auth

像这家卖场自己的办卡流程、登录方式和向导提示。

Catalog / dynamic model

像货架目录和临时上新能力，不一定每一件货都预先写死在静态清单里。

Runtime quirks

像不同供货商各自的包装习惯、运输要求、思考按钮、缓存规则。

🎈 记一句

Provider 插件不是“给一个名字”，而是“把这家店整套对接方式接进来”。

钥匙轮换

🔁 API key rotation 像“这家店给了你一串会员卡，限流了就换下一张”

官方把这个逻辑讲得很务实：不是所有错误都换卡，只有像 429 这种限流类问题才会试下一张。

🎫

<PROVIDER>_API_KEYS

像一串备用会员卡，挂在同一家卖场门口。

⚡

OPENCLAW_LIVE_..._KEY

像今天临时插队用的最高优先级特别卡。

429

只在该换时换

像门口说“这张卡今天刷太多次了”，才换下一张；不是一遇到任何问题都胡乱换卡。

📌

一句话

轮换不是乱试，是按限流这类特定信号有礼貌地换下一张卡。

常见供货商

🌟 官方后半段像在带你逛样板货架

OpenAI、Anthropic、Google、OpenRouter、OpenCode、Moonshot……这些都是样板摊位，告诉你各家大概怎么进门、默认脑子叫什么。

OpenAI / Anthropic

像最常被摆在中间展台的两家，大多数人都会先从这边起步。

Google / Z.AI / Moonshot

像新近活跃的强力货架，型号、思考方式、上下文大小各有自己的脾气。

OpenRouter / Kilo / LiteLLM

像集市型卖场，一站能摸到很多牌子的货。

📚 这页真正作用

不是让你一次背完所有货架，而是告诉你“OpenClaw 的供应链系统有多能装、有多能接”。

最后总结

🎈 把这页压成一句最短的话

Model providers 讲的不是某一颗模型，而是整套供货体系：门牌格式、上架目录、办卡方式、运行时怪脾气、钥匙轮换和插件如何把新卖场接进来。
